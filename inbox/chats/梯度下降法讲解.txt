我们从第 2 页开始。第 2 页的核心是：什么叫“迭代算法”，以及什么叫“下降方向”。这是整份讲义的起点，因为梯度下降法本质上就是一种特殊的迭代算法。讲义考虑的是无约束优化问题，也就是

[
\min_{x\in \mathbb{R}^n} f(x),
]

其中 (f) 是一个可微函数。这里“无约束”的意思是，变量 (x) 可以在整个 (\mathbb{R}^n) 中取值，没有类似 (x\ge 0)、(Ax=b)、(x\in C) 这样的额外限制。可微则意味着我们可以在每一点计算梯度 (\nabla f(x))，也就是函数在该点附近变化最快的方向信息。

所谓迭代算法，就是我们不试图一次性直接写出最优解 (x^\ast)，而是从一个初始点 (x_0) 出发，一步一步地产生点列

[
x_0,\ x_1,\ x_2,\ \ldots
]

希望这个点列逐渐靠近最优解。讲义中给出的通用迭代格式是

[
x_{k+1}=x_k+t_kp_k,\qquad k=0,1,2,\ldots
]

这一个式子非常重要。它说明第 (k+1) 个点是从当前点 (x_k) 出发，沿着某个方向 (p_k)，走一段长度 (t_k) 得到的。这里 (p_k) 叫搜索方向，(t_k) 叫步长。你可以把 (p_k) 理解为“往哪里走”，把 (t_k) 理解为“走多远”。

比如在二维平面中，如果当前点是 (x_k=(1,2))，方向是 (p_k=(-1,0))，步长是 (t_k=0.3)，那么下一步就是

[
x_{k+1}=(1,2)+0.3(-1,0)=(0.7,2).
]

这说明我们从 ((1,2)) 向左走了 (0.3) 个单位。优化算法的关键就在于：方向 (p_k) 怎么选，步长 (t_k) 怎么选。

接下来讲义引入“下降方向”。如果

[
p_k^T\nabla f(x_k)<0,
]

那么 (p_k) 就叫做 (f) 在 (x_k) 处的下降方向。这个定义不是凭空来的，它来自一阶泰勒展开。因为 (f) 可微，所以当 (t) 很小时，

[
f(x_k+tp_k)\approx f(x_k)+t,\nabla f(x_k)^Tp_k.
]

注意这里的 (\nabla f(x_k)^Tp_k) 是方向导数，也就是函数沿着 (p_k) 方向的瞬时变化率。如果它小于零，那么对于足够小的正步长 (t>0)，右边的变化量

[
t,\nabla f(x_k)^Tp_k
]

也是负数，所以

[
f(x_k+tp_k)<f(x_k)
]

至少在局部近似意义下成立。这就是为什么条件 (p_k^T\nabla f(x_k)<0) 表示“沿着 (p_k) 走会让函数值下降”。

这里要特别注意符号。梯度 (\nabla f(x_k)) 指向的是函数上升最快的方向，而我们做最小化问题，希望函数值下降，所以自然会想到往负梯度方向走，也就是

[
p_k=-\nabla f(x_k).
]

把这个方向代入下降方向条件，有

[
p_k^T\nabla f(x_k)
==================

# (-\nabla f(x_k))^T\nabla f(x_k)

-|\nabla f(x_k)|^2.
]

只要 (\nabla f(x_k)\ne 0)，就有

[
-|\nabla f(x_k)|^2<0.
]

因此负梯度方向一定是下降方向。这也是后面梯度下降法的根本依据。

我们再把它解释得更直观一点。假设你站在一座山坡上，函数值 (f(x)) 表示高度。梯度 (\nabla f(x)) 指向最陡的上坡方向，所以 (-\nabla f(x)) 就指向最陡的下坡方向。如果你想下山，最自然的局部选择就是沿着 (-\nabla f(x)) 走。不过这里说的是“局部最陡”，不是全局最优，因为你只能通过当前点附近的坡度判断下一步，不知道远处地形如何。

这一页其实已经包含了梯度下降法的骨架。一般迭代算法是

[
x_{k+1}=x_k+t_kp_k.
]

如果我们进一步取

[
p_k=-\nabla f(x_k),
]

就得到梯度下降法的基本形式：

[
x_{k+1}=x_k-t_k\nabla f(x_k).
]

现在你需要先牢牢记住三个概念：第一，(x_k) 是当前点；第二，(p_k) 是从当前点出发的搜索方向；第三，(t_k) 是沿这个方向走的步长。判断一个方向是不是下降方向，只看它和梯度的内积是否为负，即 (p_k^T\nabla f(x_k)<0)。

在继续第 3 页之前，你可以先检查自己是否理解下面这个小问题：为什么 (-\nabla f(x_k)) 一定是下降方向？如果你能说出“因为 ((-\nabla f(x_k))^T\nabla f(x_k)=-|\nabla f(x_k)|^2<0)”这个理由，就说明第 2 页的核心已经掌握了。

好，我们继续第 3 页和第 4 页。第 3 页是一个直观比喻，第 4 页开始正式给出经典梯度下降法，并且给出一个非常重要的“近端解释”。这部分虽然看起来简单，但它会帮助你理解为什么梯度下降不是随便写出来的公式，而是有明确优化意义的算法。

第 3 页用的是“雾中下山”的类比。你被困在一座山上，周围全是雾，看不见整座山的地形，只能感受到脚下地面的坡度。你的目标是到达山谷最低点，也就是优化问题里的最小点 (x^\ast)。这个比喻对应到数学中，就是我们想求

[
\min_x f(x),
]

但是在当前位置 (x_k)，我们一般不知道整个函数的全局形状，只知道局部信息，尤其是当前点的函数值 (f(x_k)) 和梯度 (\nabla f(x_k))。第 3 页强调的第一件事是方向 (p_k)。在雾中，你会选择脚下最陡的下坡方向；在数学中，这个方向就是负梯度方向

[
-\nabla f(x_k).
]

第二件事是步长 (t_k)。方向只告诉你往哪里走，但不告诉你走多远。即使方向是正确的，如果步长太小，下降会很慢；如果步长太大，可能会越过谷底，甚至跑到更高的地方。第三件事是重复迭代。你走到新位置后，再重新感受当地坡度，再选新的下坡方向，再走一步。这样得到一串点 (x_0,x_1,x_2,\ldots)，希望最终来到一个很平的地方，也就是

[
\nabla f(x_k)\approx 0.
]

这里要注意，“很平”不一定就是全局最低点。对于一般非凸函数，梯度接近零可能是局部最小点，也可能是鞍点，甚至是局部最大点。不过在梯度下降法的基础讲解里，我们先把它理解为“算法没有明显下降方向了”。第 3 页的作用就是建立直觉：梯度下降法是一种只依赖局部坡度信息的下山方法。

现在进入第 4 页。经典梯度下降法取

[
p_k=-\nabla f(x_k),
]

于是通用迭代格式

[
x_{k+1}=x_k+t_kp_k
]

变成

[
x_{k+1}=x_k-t_k\nabla f(x_k).
]

这就是梯度下降法最核心的更新公式。每一步先计算当前位置的梯度，然后沿负梯度方向走一步。这里 (t_k>0) 是步长，也常被叫做学习率。

第 4 页还说，(-\nabla f(x_k)) 也叫最速下降方向。这个说法需要稍微解释一下。假设我们只允许走一个很短的单位方向 (d)，并且规定 (|d|=1)。根据一阶近似，

[
f(x_k+\epsilon d)\approx f(x_k)+\epsilon \nabla f(x_k)^Td.
]

为了让函数值下降得最快，我们希望 (\nabla f(x_k)^Td) 尽可能小。由 Cauchy-Schwarz 不等式，

[
\nabla f(x_k)^Td\ge -|\nabla f(x_k)|,|d|.
]

因为 (|d|=1)，所以

[
\nabla f(x_k)^Td\ge -|\nabla f(x_k)|.
]

等号在

[
d=-\frac{\nabla f(x_k)}{|\nabla f(x_k)|}
]

时取到。因此，在欧氏距离意义下，最陡的下降方向就是负梯度方向。注意严格说，“最速下降方向”依赖于我们使用的范数；在这份讲义里默认使用的是欧氏范数，所以负梯度就是最速下降方向。

第 4 页最重要、也最值得认真理解的是下面这个解释：

[
x_{k+1}
=======

\arg\min_x
\left[
f(x_k)+\nabla f(x_k)^T(x-x_k)+\frac{1}{2t_k}|x-x_k|^2
\right].
]

这个式子看起来比普通更新公式复杂，但它其实在告诉我们：梯度下降的下一步 (x_{k+1})，可以看成是在最小化一个“局部近似模型”。

我们逐项解释。第一项

[
f(x_k)
]

是当前点的函数值。对于变量 (x) 来说，这是常数，不影响最小化的位置。第二项

[
\nabla f(x_k)^T(x-x_k)
]

是函数在 (x_k) 处的一阶线性近似。也就是说，我们用当前点的切平面来近似真实函数。只看这两项，就是

[
f(x_k)+\nabla f(x_k)^T(x-x_k).
]

但是这个线性函数有一个问题：如果 (\nabla f(x_k)\ne 0)，它通常没有最小值。比如沿着负梯度方向一直走，线性模型会一直下降到 (-\infty)。这显然不合理，因为真实函数不一定一直下降，而且线性近似只在 (x_k) 附近可靠。

所以讲义加上了第三项

[
\frac{1}{2t_k}|x-x_k|^2.
]

这一项叫近端项，或者正则化项。它的作用是惩罚 (x) 离当前点 (x_k) 太远。换句话说，我们希望下一点既能让线性近似下降，又不要离当前点太远，因为离太远以后当前梯度信息就不可靠了。

现在我们来精确推导这个最小化问题为什么会得到梯度下降公式。令

[
Q(x)=f(x_k)+\nabla f(x_k)^T(x-x_k)+\frac{1}{2t_k}|x-x_k|^2.
]

我们要求

[
x_{k+1}=\arg\min_x Q(x).
]

对 (Q(x)) 关于 (x) 求梯度。第一项 (f(x_k)) 是常数，梯度为零。第二项的梯度是

[
\nabla f(x_k).
]

第三项的梯度是

[
\frac{1}{t_k}(x-x_k).
]

所以

[
\nabla Q(x)=\nabla f(x_k)+\frac{1}{t_k}(x-x_k).
]

令一阶最优条件成立：

[
\nabla Q(x)=0.
]

于是

[
\nabla f(x_k)+\frac{1}{t_k}(x-x_k)=0.
]

移项得到

[
\frac{1}{t_k}(x-x_k)=-\nabla f(x_k),
]

两边乘以 (t_k)，得到

[
x-x_k=-t_k\nabla f(x_k).
]

因此

[
x=x_k-t_k\nabla f(x_k).
]

这正是梯度下降更新公式。所以第 4 页这个式子说明，梯度下降不仅仅是“往负梯度方向走”，它还等价于在当前点附近构造一个“线性近似加二次惩罚”的简单模型，然后最小化这个模型。

这个解释非常重要，因为它连接了很多后续算法。比如近端梯度法、镜像下降、牛顿法、信赖域方法，都可以从“构造局部模型然后最小化”这个角度理解。现在你可以先把它理解成一句话：梯度下降每一步是在最小化一个容易处理的局部二次模型。

第 4 页最后给出算法流程。先选初始点 (x_0)，然后对 (k=0,1,2,\ldots) 重复：选择合适步长 (t_k)，更新

[
x_{k+1}=x_k-t_k\nabla f(x_k),
]

如果满足停止条件就停下。这里目前还没有讲什么叫“合适步长”，因为第 5 页和第 6 页会专门讨论停止准则和步长规则。

这一部分你需要掌握的核心是：梯度下降公式来自两种等价理解。第一种是几何直觉：负梯度是局部最陡下降方向。第二种是优化模型：下一步是最小化“当前函数的一阶近似加上不要走太远的二次惩罚项”。尤其是刚才的推导你要能自己写出来，也就是从

[
\nabla f(x_k)+\frac{1}{t_k}(x-x_k)=0
]

推出

[
x_{k+1}=x_k-t_k\nabla f(x_k).
]

如果这一页也没问题，我们下一次就进入第 5 页，讲实际算法什么时候停止，也就是 termination criteria。

我们继续第 5 页，主题是停止准则，也就是梯度下降算法运行到什么时候应该停下来。这个问题看起来像工程细节，但其实非常重要，因为理论上我们常说最优点满足一阶条件

[
\nabla f(x^\ast)=0,
]

可是计算机做数值计算时几乎不可能真的得到一个梯度完全等于零的点。讲义第 5 页开头说，条件

[
|\nabla f(x)|=0
]

在有限精度计算中并不实用，原因就在这里。计算机中的小数是浮点数，有舍入误差；而且迭代算法通常是逐步逼近最优点，不是一步精确命中最优点。所以实际中我们不会要求梯度范数严格等于零，而是要求它“足够小”。

第一个停止准则是梯度小：

[
|\nabla f(x_k)|\le \epsilon.
]

这个准则最直接。因为对于无约束可微优化问题，如果 (x^\ast) 是局部最优点，通常至少要满足一阶必要条件 (\nabla f(x^\ast)=0)。所以当 (|\nabla f(x_k)|) 已经很小时，我们认为当前点接近一个驻点。这里的 (\epsilon) 是容差，比如 (10^{-6})、(10^{-8}) 等。容差越小，要求越严格，通常需要更多迭代。

但是这个准则也有局限。第一，如果函数缩放很大，梯度可能也很大；如果函数缩放很小，梯度可能天然很小。第二，对于一些病态问题，梯度很小不一定代表离最优点很近。第三，对于非凸函数，梯度小只说明接近驻点，不保证是全局最小点。因此实际算法往往会同时使用多个停止准则。

第二个停止准则是相邻两次函数值变化很小：

[
|f(x_{k+1})-f(x_k)|\le \epsilon.
]

它的意思是，如果这一步走完以后目标函数几乎没有下降，那么继续迭代可能收益不大。举个直观例子，如果上一轮函数值是 (100.000001)，下一轮是 (100.0000005)，变化只有 (5\times 10^{-7})，而你的容忍误差是 (10^{-6})，那就可以认为目标值已经基本稳定。

不过这个绝对变化准则也有问题。如果函数值本身很大，比如 (f(x_k)=10^{10})，那么变化 (10^{-3}) 从绝对值看似乎不小，但相对 (10^{10}) 来说几乎可以忽略。反过来，如果函数值本身很小，比如 (f(x_k)=10^{-12})，那么一个绝对变化 (10^{-8}) 反而很大。因此讲义给出相对形式：

[
\frac{|f(x_{k+1})-f(x_k)|}{|f(x_k)|}\le \epsilon.
]

这个式子衡量的是函数值变化占当前函数值规模的比例。比如 (f(x_k)=1000)，变化 (0.001)，相对变化就是 (10^{-6})。这样比单纯看绝对变化更能适应不同尺度的问题。

第三个停止准则是相邻两次迭代点的距离很小：

[
|x_{k+1}-x_k|\le \epsilon.
]

它表示算法的步子已经很小，当前位置几乎不再变化。对于梯度下降法，由于

[
x_{k+1}-x_k=-t_k\nabla f(x_k),
]

所以如果步长 (t_k) 不是异常小，那么 (|x_{k+1}-x_k|) 小通常也暗示梯度小。但这两个准则不完全等价，因为如果线搜索把步长选得很小，即使梯度还不小，点差也可能很小。所以它可以作为辅助判断，但单独使用时要谨慎。

类似地，讲义也给出相对点差准则：

[
\frac{|x_{k+1}-x_k|}{|x_k|}\le \epsilon.
]

它衡量的是这一步移动距离相对于当前点规模有多大。如果 (x_k) 的范数很大，那么一个绝对距离 (10^{-3}) 可能完全不重要；如果 (x_k) 本身很接近零，同样的移动距离就可能很显著。

最后一条是数值稳定性问题。讲义说，为了避免除以很小的数造成不稳定，可以把相对准则中的分母替换为

[
\max{1,|f(x_k)|}
]

和

[
\max{1,|x_k|}.
]

这是什么意思？以函数值相对变化为例，原来的形式是

[
\frac{|f(x_{k+1})-f(x_k)|}{|f(x_k)|}\le \epsilon.
]

如果 (|f(x_k)|) 非常小，比如 (10^{-12})，那么分母太小，会导致比值被人为放大，甚至出现数值不稳定。于是我们改成

[
\frac{|f(x_{k+1})-f(x_k)|}{\max{1,|f(x_k)|}}\le \epsilon.
]

这样当 (|f(x_k)|<1) 时，分母直接取 (1)，避免除以一个接近零的数；当 (|f(x_k)|\ge 1) 时，分母仍然是 (|f(x_k)|)，保留相对变化的意义。

点差准则同理，把

[
\frac{|x_{k+1}-x_k|}{|x_k|}
]

改成

[
\frac{|x_{k+1}-x_k|}{\max{1,|x_k|}}.
]

这是一种非常常见的数值优化写法。它兼顾了绝对误差和相对误差：当量本身很小时，用绝对尺度判断；当量本身很大时，用相对尺度判断。

总结一下，第 5 页讲的是算法停止的实际标准。最核心的有三类：梯度是否足够小，函数值是否几乎不再下降，迭代点是否几乎不再移动。其中梯度小最接近理论上的一阶最优条件；函数值变化小反映目标值稳定；点差小反映变量稳定。实际实现中常常会组合使用这些条件，比如只要梯度足够小就停，或者函数值变化和点差都很小就停，同时还会设置最大迭代次数，避免算法无限运行。

你现在需要特别分清楚这三个量分别在看什么：

[
|\nabla f(x_k)|
]

看的是“当前位置还陡不陡”；

[
|f(x_{k+1})-f(x_k)|
]

看的是“目标值还降不降”；

[
|x_{k+1}-x_k|
]

看的是“点还动不动”。

如果你理解了这三者的区别和为什么要用 (\max{1,\cdot}) 避免除以很小的数，我们下一步就进入第 6 页，讲步长规则，也就是梯度下降中最关键的“到底走多远”。


这很好理解，请继续。


我们继续第 6 页，主题是步长规则。前面我们已经知道梯度下降的方向通常取

[
p_k=-\nabla f(x_k),
]

所以更新公式是

[
x_{k+1}=x_k-t_k\nabla f(x_k).
]

现在问题就变成：方向已经定了，步长 (t_k) 应该怎么选？第 6 页列出三种典型方法：常数步长、精确线搜索和回溯线搜索。其中回溯线搜索是讲义特别强调“最实用”的方法。

先讲常数步长。常数步长就是每一步都取同一个正数

[
t_k=\alpha,\qquad \alpha>0.
]

于是迭代变成

[
x_{k+1}=x_k-\alpha \nabla f(x_k).
]

这种方法最简单，尤其在机器学习里经常把 (\alpha) 称为 learning rate，也就是学习率。常数步长的优点是实现方便，每一步不用额外计算；缺点是 (\alpha) 很难选。若 (\alpha) 太小，每一步下降很慢；若 (\alpha) 太大，可能越过最优点，导致震荡，甚至函数值上升或发散。

用一维例子可以很直观看到这一点。假设

[
f(x)=x^2.
]

那么

[
\nabla f(x)=2x.
]

梯度下降为

[
x_{k+1}=x_k-2\alpha x_k=(1-2\alpha)x_k.
]

如果 (0<\alpha<1)，那么 (|1-2\alpha|<1)，点会逐渐趋近于 (0)。如果 (\alpha=\frac12)，一步就到 (0)。如果 (\alpha>1)，那么 (|1-2\alpha|>1)，迭代点反而越来越远，算法发散。这个简单例子说明步长不是越大越好。

第二种是精确线搜索。讲义写作

[
t_k=\arg\min_{t\ge 0} f(x_k+t p_k).
]

这句话的意思是：当前点 (x_k) 和方向 (p_k) 已经确定了，我们只在这条射线上找最好的步长。换句话说，把多维问题暂时变成一个一维问题。定义

[
\phi(t)=f(x_k+t p_k),
]

那么精确线搜索就是求

[
t_k=\arg\min_{t\ge 0}\phi(t).
]

如果是梯度下降方向 (p_k=-\nabla f(x_k))，就是在负梯度方向这条线上找让函数值最小的那个点。

精确线搜索听起来很理想，因为每一步都沿当前方向走到最优位置。但它通常不实用。原因是，求

[
\arg\min_{t\ge 0} f(x_k+t p_k)
]

本身可能就是一个不简单的一维优化问题。尤其当 (f) 很复杂时，每次迭代都精确求最优步长，代价可能很高。更重要的是，即使每一步在当前方向上最优，也不代表整体收敛最快。后面第 7、8 页会看到，精确线搜索在狭长椭圆等高线中会产生“之字形”运动，仍然可能很慢。

第三种是回溯线搜索，也叫 backtracking line search。第 6 页暂时只列出名字，第 9 页会正式讲 Armijo 规则。我们先建立直觉。回溯线搜索不是一上来就精确求最优步长，而是先尝试一个较大的步长，比如 (t=1)。如果这一步让函数值下降得足够多，就接受；如果下降不够，就把步长缩小，比如乘以 (\beta=\frac12)，再试一次。如此反复，直到找到一个令人满意的步长。

所以三种步长策略可以这样理解。常数步长是“我永远走固定长度”；精确线搜索是“我在当前方向上精确找到最好的长度”；回溯线搜索是“我从一个大胆的长度开始试，如果不合适就逐渐缩短，直到足够好”。在实际优化中，回溯线搜索常常比精确线搜索更实用，因为它不追求一步方向上的完美，只要求足够下降，从而减少计算代价并提高鲁棒性。

第 6 页还总结了梯度下降法的优点和缺点。优点之一是每次迭代便宜。因为每一步主要只需要计算梯度 (\nabla f(x_k))，然后做一次向量更新

[
x_{k+1}=x_k-t_k\nabla f(x_k).
]

如果 (f) 的梯度容易计算，那么单步成本通常不高。相比之下，牛顿法需要二阶导数，也就是 Hessian 矩阵 (\nabla^2 f(x_k))，还需要解线性方程，单步成本通常更高。

第二个优点是梯度下降不需要二阶导数。很多问题中，一阶导数比较容易得到，二阶导数要么太复杂，要么存储代价太高。比如在高维机器学习问题中，变量维度可能有几万、几百万，Hessian 是一个巨大的矩阵，直接计算和存储都很困难。梯度下降只用梯度，因此更适合大规模问题。

但是梯度下降的主要缺点是：在病态问题或者条件数很大的问题上收敛很慢。所谓病态问题，直观上就是函数的等高线很狭长，有些方向很陡，有些方向很平。梯度下降每一步只看最陡下降方向，容易在狭长谷底两侧来回摆动，而不是沿着谷底快速前进。这个问题后面第 18 到第 20 页会从条件数角度系统解释。

我们现在把第 6 页的逻辑串起来。梯度下降法的方向通常已经确定为负梯度，但步长决定了算法实际表现。常数步长简单但需要谨慎选择；精确线搜索理论上自然但实际可能昂贵；回溯线搜索在实践中常用，因为它在计算成本和下降效果之间取得平衡。梯度下降的优势是单步便宜、不需要 Hessian；劣势是在缩放不好、条件数大的问题上可能非常慢。

为了确保你真正掌握这一页，我给你一个判断标准：如果你能解释为什么“精确线搜索每一步看起来最优，但整个算法仍可能慢”，那你就已经抓住了重点。答案是，因为它只是在当前方向上最优，而当前方向本身可能不好；如果函数等高线狭长，负梯度方向常常指向谷壁而不是谷底方向，于是算法会之字形前进。

好，我们进入第 7 页。第 7 页开始讨论精确线搜索，特别是在二次函数上的精确步长公式。这个推导很重要，因为它不仅能让你明白线搜索怎么算，还会解释后面为什么梯度下降在某些问题上会“之字形”缓慢前进。讲义这一页考虑的二次函数形式是

[
f(x)=x^TAx+2b^Tx+c,\qquad A\succ 0.
]

这里 (A\succ 0) 表示 (A) 是对称正定矩阵。正定意味着对任意非零向量 (z)，都有

[
z^TAz>0.
]

这保证了这个二次函数是严格凸的，因此有唯一全局最小点。

先求梯度。对于

[
f(x)=x^TAx+2b^Tx+c,
]

如果 (A) 是对称矩阵，那么

[
\nabla f(x)=2Ax+2b.
]

令梯度为零，可以得到最优点满足

[
2Ax^\ast+2b=0,
]

所以

[
Ax^\ast=-b,
]

也就是

[
x^\ast=-A^{-1}b.
]

不过在梯度下降中，我们通常不是直接解这个线性方程，而是从某个 (x_k) 出发，沿方向 (p_k) 搜索：

[
x_{k+1}=x_k+t_kp_k.
]

精确线搜索要做的是，在当前方向 (p_k) 上选择最好的 (t_k)，也就是

[
t_k=\arg\min_{t\ge 0} f(x_k+tp_k).
]

为了推导这个 (t_k)，我们把

[
\phi(t)=f(x_k+tp_k)
]

看成关于单变量 (t) 的函数。将 (x_k+tp_k) 代入二次函数，有

[
\phi(t)
=======

(x_k+tp_k)^TA(x_k+tp_k)+2b^T(x_k+tp_k)+c.
]

现在逐项展开。第一项是

[
(x_k+tp_k)^TA(x_k+tp_k)
=======================

x_k^TAx_k+2t p_k^TAx_k+t^2p_k^TAp_k,
]

其中用到了 (A) 对称，所以

[
x_k^TAp_k=p_k^TAx_k.
]

第二项是

[
2b^T(x_k+tp_k)=2b^Tx_k+2t b^Tp_k.
]

所以

[
\phi(t)
=======

x_k^TAx_k+2b^Tx_k+c
+
2t p_k^TAx_k+2t b^Tp_k
+
t^2p_k^TAp_k.
]

把常数项合在一起，它就是 (f(x_k))。把一次项也合在一起：

[
2t(p_k^TAx_k+b^Tp_k)
====================

2t p_k^T(Ax_k+b).
]

而因为

[
\nabla f(x_k)=2Ax_k+2b=2(Ax_k+b),
]

所以

[
p_k^T\nabla f(x_k)=2p_k^T(Ax_k+b).
]

因此一次项可以写成

[
t p_k^T\nabla f(x_k).
]

于是

[
\phi(t)
=======

f(x_k)+t p_k^T\nabla f(x_k)+t^2p_k^TAp_k.
]

这是一个关于 (t) 的一元二次函数。因为 (A\succ 0)，只要 (p_k\ne 0)，就有

[
p_k^TAp_k>0,
]

所以它是一个开口向上的抛物线，有唯一最小点。

现在对 (t) 求导：

[
\phi'(t)=p_k^T\nabla f(x_k)+2t p_k^TAp_k.
]

精确线搜索的最优步长满足

[
\phi'(t)=0.
]

所以

[
p_k^T\nabla f(x_k)+2t p_k^TAp_k=0.
]

解得

[
t_k
===

-\frac{p_k^T\nabla f(x_k)}{2p_k^TAp_k}.
]

这就是针对讲义中二次函数 (x^TAx+2b^Tx+c) 的精确线搜索步长公式。注意分母是正的，因为 (A\succ0)；如果 (p_k) 是下降方向，那么

[
p_k^T\nabla f(x_k)<0,
]

所以分子前面有负号，最终 (t_k>0)。这很合理，因为下降方向对应的最优步长应该是正数。

如果采用梯度下降方向

[
p_k=-\nabla f(x_k),
]

那么公式变成

[
t_k
===

\frac{\nabla f(x_k)^T\nabla f(x_k)}
{2\nabla f(x_k)^TA\nabla f(x_k)}.
]

这里要小心一个系数。因为讲义这一页的二次函数写作

[
x^TAx+2b^Tx+c,
]

所以梯度是 (2Ax+2b)，精确线搜索公式里分母有 (2p_k^TAp_k)。如果二次函数写成更常见的形式

[
f(x)=\frac12 x^TAx+b^Tx+c,
]

那么梯度是

[
\nabla f(x)=Ax+b,
]

对应的精确线搜索公式会变成

[
t_k
===

-\frac{p_k^T\nabla f(x_k)}{p_k^TAp_k}.
]

所以看到不同教材中公式差一个系数，不要慌，通常只是二次函数前面的系数写法不同。

现在看第 7 页的例子。讲义给出

[
f(x)=\frac12\left((x^{(1)})^2+\gamma (x^{(2)})^2\right),\qquad \gamma\ge 1,
]

从初始点

[
x_0=(\gamma,1)
]

出发，并使用精确线搜索。这个函数的最优点显然是

[
x^\ast=(0,0),
]

因为两个平方项都非负，只有在两个坐标都为零时函数值最小。

这个函数的梯度是

[
\nabla f(x)=
\begin{pmatrix}
x^{(1)}\
\gamma x^{(2)}
\end{pmatrix}.
]

如果 (\gamma=1)，那么函数就是一个圆形碗：

[
f(x)=\frac12\left((x^{(1)})^2+(x^{(2)})^2\right).
]

此时等高线是圆，负梯度方向直接指向原点，梯度下降会非常快。但如果 (\gamma) 很大，比如 (\gamma=10000)，第二个方向的曲率极大，等高线会变成非常狭长的椭圆。这时负梯度方向会被陡峭方向强烈影响，导致算法不直接朝原点走，而是在狭长谷地两边来回折返。

讲义给出了一个误差收缩公式：

[
\frac{|x_k-x^\ast|_2}{|x_0-x^\ast|_2}
=====================================

\left(\frac{\gamma-1}{\gamma+1}\right)^k.
]

这个公式告诉我们，当 (\gamma) 越大时，

[
\frac{\gamma-1}{\gamma+1}
]

越接近 (1)，于是误差下降越慢。比如 (\gamma=10000) 时，

[
\frac{\gamma-1}{\gamma+1}
=========================

\frac{9999}{10001}
\approx 0.9998.
]

每一步误差大约只乘以 (0.9998)，下降得非常慢。这就是所谓病态问题或条件数大导致梯度下降缓慢的典型例子。

我们稍微解释一下这个比例为什么和 (\gamma) 有关。这个函数的 Hessian 是

[
\nabla^2 f(x)=
\begin{pmatrix}
1&0\
0&\gamma
\end{pmatrix}.
]

它的最小特征值是 (1)，最大特征值是 (\gamma)，所以条件数是

[
\kappa=\gamma.
]

当 (\gamma) 大时，说明不同方向上的曲率差异巨大：一个方向比较平，另一个方向非常陡。梯度下降必须选一个全局步长，既不能在陡方向上走太大导致震荡，又要在平方向上取得进展。这个矛盾会让算法非常慢。虽然“条件数”在后面第 18 页才正式讲，但这个例子已经提前展示了它的影响。

这一页你需要掌握两件事。第一，二次函数精确线搜索的推导方法：把多维函数限制到直线 (x_k+tp_k) 上，得到一元函数 (\phi(t))，再令 (\phi'(t)=0)。第二，精确线搜索并不等于算法一定快，因为它只是沿当前方向选最优步长；如果当前方向由于函数缩放不好而反复偏向谷壁，那么每一步虽然在线上最优，但整体路径仍然很曲折。

接下来我们会讲第 8 页。第 8 页用图像展示

[
f(x,y)=100x^2+y^2
]

从初始点 ((1,100)) 出发时的运动轨迹，并证明精确线搜索梯度下降中的相邻两步方向互相正交，也就是

[
(x_{k+2}-x_{k+1})^T(x_{k+1}-x_k)=0.
]

这个性质正是“之字形运动”的数学原因。


我理解了精确线搜索的推导和劣势，请你继续讲解。


我们继续第 8 页。这一页的主题是：为什么梯度下降法在某些二次函数上会出现“之字形”运动，并且为什么这种运动会导致收敛很慢。讲义给出的例子是

[
f(x,y)=100x^2+y^2,
]

初始点是

[
(1,100).
]

第 8 页右侧的图展示了这个函数的等高线和梯度下降轨迹。等高线是一系列非常狭长的椭圆，而不是接近圆形的曲线。红色折线表示梯度下降的迭代路径，可以看到它不是直接向最优点走，而是在椭圆谷地两侧来回折返，这就是所谓的 zig-zag motion，也就是“之字形”运动。

先看这个函数本身。它的最优点显然是

[
(x^\ast,y^\ast)=(0,0),
]

因为 (100x^2\ge 0)，(y^2\ge 0)，只有 (x=0,y=0) 时函数值最小。它的梯度是

[
\nabla f(x,y)=
\begin{pmatrix}
200x\
2y
\end{pmatrix}.
]

注意两个方向上的系数差别非常大。(x) 方向的梯度分量是 (200x)，而 (y) 方向的梯度分量是 (2y)。这说明函数在 (x) 方向非常陡，在 (y) 方向相对平缓。换成几何语言，函数像一个狭长山谷：横向很陡，纵向很平。

从初始点 ((1,100)) 出发，梯度是

[
\nabla f(1,100)=
\begin{pmatrix}
200\
200
\end{pmatrix}.
]

负梯度方向是

[
-\nabla f(1,100)=
\begin{pmatrix}
-200\
-200
\end{pmatrix}.
]

这个方向看上去是往左下方走。但是最优点 ((0,0)) 相对于 ((1,100)) 的方向是

[
(0,0)-(1,100)=(-1,-100),
]

它几乎是竖直向下的方向。也就是说，负梯度方向并没有直接指向最优点，而是明显偏向 (x) 方向。这正是因为 (x) 方向太陡，梯度被陡峭方向主导了。

这件事非常关键。梯度指向函数值上升最快的方向，负梯度指向函数值下降最快的局部方向；但在狭长谷地里，“局部最陡下降方向”往往是朝向谷壁的，而不是沿着谷底通向最优点的方向。所以即使每一步都沿负梯度下降，路径也可能不是一条直线，而是不断撞向两侧谷壁，再折回来。

第 8 页还写了一个非常重要的性质：对于精确线搜索梯度下降法，有

[
(x_{k+2}-x_{k+1})^T(x_{k+1}-x_k)=0.
]

这个式子说明连续两步的位移方向互相正交。也就是说，第 (k) 步从 (x_k) 到 (x_{k+1})，下一步从 (x_{k+1}) 到 (x_{k+2})，这两段路的夹角是 (90^\circ)。这正是图中路径呈现折线状、之字形的数学原因。

我们现在来推导这个性质。梯度下降的更新是

[
x_{k+1}=x_k-t_k\nabla f(x_k).
]

因此第 (k) 步的位移是

[
x_{k+1}-x_k=-t_k\nabla f(x_k).
]

第 (k+1) 步的位移是

[
x_{k+2}-x_{k+1}=-t_{k+1}\nabla f(x_{k+1}).
]

所以要证明

[
(x_{k+2}-x_{k+1})^T(x_{k+1}-x_k)=0,
]

只需要证明

[
\nabla f(x_{k+1})^T\nabla f(x_k)=0.
]

也就是说，连续两次迭代点处的梯度互相正交。

为什么精确线搜索会导致这个性质？回忆精确线搜索的定义。在第 (k) 步，我们沿方向

[
p_k=-\nabla f(x_k)
]

寻找最优步长：

[
t_k=\arg\min_{t\ge 0} f(x_k+t p_k).
]

定义一元函数

[
\phi(t)=f(x_k+t p_k).
]

如果 (t_k) 是这个一元函数的内部最小点，那么它满足一阶条件

[
\phi'(t_k)=0.
]

根据链式法则，

[
\phi'(t)=\nabla f(x_k+t p_k)^T p_k.
]

把 (t=t_k) 代入，注意

[
x_{k+1}=x_k+t_kp_k,
]

于是得到

[
0=\phi'(t_k)=\nabla f(x_{k+1})^Tp_k.
]

又因为

[
p_k=-\nabla f(x_k),
]

所以

[
\nabla f(x_{k+1})^T(-\nabla f(x_k))=0.
]

也就是

[
\nabla f(x_{k+1})^T\nabla f(x_k)=0.
]

现在回到位移向量。因为

[
x_{k+1}-x_k=-t_k\nabla f(x_k),
]

而

[
x_{k+2}-x_{k+1}=-t_{k+1}\nabla f(x_{k+1}),
]

所以

[
(x_{k+2}-x_{k+1})^T(x_{k+1}-x_k)
================================

t_{k+1}t_k
\nabla f(x_{k+1})^T\nabla f(x_k)
================================

0.

]

于是我们证明了连续两步位移方向正交。

这个结论很有意思。精确线搜索听起来似乎很聪明，因为每一步都在当前方向上走到最优位置。但是它也带来一个几何后果：到达 (x_{k+1}) 后，新梯度必须和上一轮搜索方向正交。由于下一步又沿新的负梯度方向走，所以两步之间形成直角。这在圆形等高线中可能不是问题，但在狭长椭圆中就会表现为非常明显的来回折返。

我们可以用“狭长山谷”再解释一遍。假如你在一个很窄的山谷里，谷壁很陡，谷底很长。负梯度方向总是倾向于指向最近的谷壁，而不是沿着谷底方向前进。精确线搜索会让你沿着这个方向走到函数值在这条直线上最低的位置，这通常会把你带到谷地另一侧。下一次梯度方向又几乎指向对面的谷壁，于是你再折回来。这样虽然每一步都下降，但大量运动浪费在横向震荡上，沿谷底朝最优点前进的速度很慢。

第 8 页中 “convergence very dependent on scaling” 这句话也很重要。所谓 scaling，指的是不同变量的尺度或曲率差异。对于

[
f(x,y)=100x^2+y^2,
]

(x) 方向的曲率远大于 (y) 方向。如果我们做变量缩放，例如令

[
u=10x,\qquad v=y,
]

那么函数可以写成

[
f=;u^2+v^2,
]

这时等高线就变成圆形，梯度下降的行为会好很多。这个想法后面会发展成“预条件”思想，也就是通过改变变量尺度或使用矩阵 (P) 来改善梯度下降的方向。

所以第 8 页真正想让你理解的是：梯度下降慢，并不一定是因为每一步没有下降，而是因为下降方向的几何质量不好。在狭长椭圆等高线中，负梯度方向会被高曲率方向支配，导致算法不断正交折返。精确线搜索只能保证沿当前方向走得最好，却不能改变当前方向本身的缺陷。

你现在需要掌握第 8 页的两个关键结论。首先，函数等高线越狭长，梯度下降越容易之字形震荡；其次，对于精确线搜索梯度下降，相邻两步位移满足正交关系

[
(x_{k+2}-x_{k+1})^T(x_{k+1}-x_k)=0.
]

如果你能理解这个正交性来自线搜索的一阶条件

[
\phi'(t_k)=0,
]

这一页就掌握得很扎实了。下一页我们会讲第 9 页的 Armijo 回溯线搜索，它的目的不是找精确最优步长，而是找一个“足够下降”的步长。

我们继续第 9 页，讲 Armijo rule line search，也就是 Armijo 回溯线搜索。前面第 7、8 页说明了精确线搜索的思想：沿着当前方向，把这条线上最好的步长精确找出来。但精确线搜索有两个问题：一是计算上可能很贵，二是即使每一步在线上最优，也不一定整体收敛很快。Armijo 回溯线搜索的思路更务实：不要求找到当前方向上的最优步长，只要求找到一个能让函数值“下降得足够多”的步长。

设当前点是 (x_k)，搜索方向是 (p_k)。我们通常要求 (p_k) 是下降方向，也就是

[
p_k^T\nabla f(x_k)<0.
]

如果是梯度下降法，那么

[
p_k=-\nabla f(x_k),
]

所以

[
p_k^T\nabla f(x_k)=-|\nabla f(x_k)|^2<0
]

只要当前梯度非零。

Armijo 回溯线搜索从一个初始候选步长开始，比如讲义中写作

[
\hat t>0,
]

常用选择是

[
\hat t=1.
]

然后不断尝试缩小步长。具体来说，令

[
t_k=\hat t\beta^s,
]

其中

[
0<\beta<1,
]

而 (s) 是非负整数。也就是说，当 (s=0) 时，

[
t_k=\hat t.
]

如果这个步长不合格，就令 (s=1)，步长变成

[
t_k=\hat t\beta.
]

如果还不合格，就继续令 (s=2)，得到

[
t_k=\hat t\beta^2.
]

常见选择是

[
\beta=\frac12.
]

所以回溯线搜索就是不断把步长减半，直到满足某个下降条件。

这个下降条件就是第 9 页的核心公式：

[
f(x_k+t_kp_k)\le f(x_k)+\alpha t_kp_k^T\nabla f(x_k).
]

这里

[
\alpha
]

是一个较小的正数，通常取

[
10^{-4}.
]

这个不等式叫 Armijo 条件，也叫 sufficient decrease condition，中文可以叫“充分下降条件”。

我们现在慢慢解释这个不等式。左边

[
f(x_k+t_kp_k)
]

是真正走一步之后的新函数值。右边

[
f(x_k)+\alpha t_kp_k^T\nabla f(x_k)
]

是一个允许的上界。因为 (p_k) 是下降方向，所以

[
p_k^T\nabla f(x_k)<0.
]

又因为 (\alpha>0)、(t_k>0)，所以

[
\alpha t_kp_k^T\nabla f(x_k)<0.
]

因此右边其实比 (f(x_k)) 小。也就是说，Armijo 条件要求

[
f(x_k+t_kp_k)
]

不仅要小于原来的 (f(x_k))，还要小到一定程度。它不是只要求下降，而是要求“下降得足够多”。

为什么右边长这样？我们回忆一阶泰勒展开：

[
f(x_k+t p_k)\approx f(x_k)+t p_k^T\nabla f(x_k).
]

这表示如果 (t) 很小，函数值的线性预测下降量大约是

[
-tp_k^T\nabla f(x_k).
]

由于 (p_k^T\nabla f(x_k)<0)，这个下降量是正的。Armijo 条件没有要求真实下降量达到完整的一阶预测下降量，而只要求达到其中的一小部分。因为右边用了

[
\alpha t p_k^T\nabla f(x_k),
]

而 (\alpha) 通常很小，比如 (10^{-4})。这意味着只要真实函数值比当前值下降了线性预测下降量的一小部分，就认为这一步可以接受。

我们可以把 Armijo 条件改写成更直观的形式。原式是

[
f(x_k+t_kp_k)\le f(x_k)+\alpha t_kp_k^T\nabla f(x_k).
]

两边移项得到

[
f(x_k)-f(x_k+t_kp_k)\ge -\alpha t_kp_k^T\nabla f(x_k).
]

左边是真实下降量，右边是要求的最低下降量。因为

[
-p_k^T\nabla f(x_k)>0,
]

所以右边是正数。也就是说，Armijo 条件要求：

[
\text{真实下降量}\ge \alpha\times \text{一阶预测下降量}.
]

这就是“充分下降”的含义。

现在看为什么回溯线搜索通常一定能找到这样的步长。由于 (p_k) 是下降方向，有

[
p_k^T\nabla f(x_k)<0.
]

当 (t) 足够小时，由可微性可知

[
f(x_k+tp_k)=f(x_k)+t p_k^T\nabla f(x_k)+o(t).
]

Armijo 右边是

[
f(x_k)+\alpha t p_k^T\nabla f(x_k).
]

因为

[
0<\alpha<1,
]

并且

[
p_k^T\nabla f(x_k)<0,
]

所以

[
t p_k^T\nabla f(x_k)
<
\alpha t p_k^T\nabla f(x_k).
]

注意这里不等号方向容易弄错，因为 (p_k^T\nabla f(x_k)) 是负数。完整的一阶线性下降项 (t p_k^T\nabla f(x_k)) 更负，而乘上较小的 (\alpha) 后，(\alpha t p_k^T\nabla f(x_k)) 没那么负。因此当 (t) 很小时，真实函数值近似于更低的线性预测值，就会低于 Armijo 允许的上界。所以只要不断把 (t) 缩小，最终会满足 Armijo 条件。

第 9 页图中的意思也正是这个。图上有一条蓝色曲线，表示沿搜索方向的一维函数

[
f(x_k+tp_k)
]

随 (t) 变化的真实值；还有一条斜直线，表示 Armijo 条件中的右端

[
f(x_k)+\alpha t p_k^T\nabla f(x_k).
]

由于 (p_k^T\nabla f(x_k)<0)，这条直线向下倾斜。只要蓝色曲线落在这条直线下方，就说明这个步长满足充分下降条件。回溯线搜索就是从较大的 (t) 开始试，如果蓝色曲线在直线上方，说明下降不够，就缩小 (t)，直到落到直线下方。

这里有一个很重要的直觉：步长太大时，函数的非线性部分可能很明显，真实函数值可能没有线性模型预测得那么低，甚至可能上升。步长变小后，线性近似更可靠，因此 Armijo 条件更容易满足。所以回溯线搜索是一个很稳健的方法。

我们把算法过程完整说一遍。在第 (k) 次迭代中，先给定下降方向 (p_k)，例如负梯度方向。然后设初始步长 (t=\hat t)。检查

[
f(x_k+t p_k)\le f(x_k)+\alpha t p_k^T\nabla f(x_k)
]

是否成立。若成立，就接受这个 (t)，令

[
x_{k+1}=x_k+t p_k.
]

若不成立，就把

[
t
]

换成

[
\beta t,
]

也就是缩小为原来的 (\beta) 倍，再检查一次。重复这个过程，直到条件成立。

和精确线搜索相比，Armijo 回溯线搜索有一个本质差别：精确线搜索试图求

[
\arg\min_{t\ge 0} f(x_k+tp_k),
]

也就是找到当前方向上的最佳点；而 Armijo 只问当前步长是否足够好。它不要求最好，只要求合格。正因为要求低，它通常计算更便宜，也更容易在实际问题中使用。

这里还要强调参数 (\alpha) 和 (\beta) 的意义。(\alpha) 控制“下降要求有多严格”。如果 (\alpha) 很大，比如接近 (1)，那么算法要求真实下降量接近完整的一阶预测下降量，条件会比较难满足，步长可能被缩得很小。如果 (\alpha) 很小，比如 (10^{-4})，只要求一点点充分下降，条件比较容易满足。实际中 (\alpha) 通常取很小。

(\beta) 控制步长缩小的速度。如果 (\beta=\frac12)，每次减半，这是最常见的选择。如果 (\beta) 更小，比如 (0.1)，步长会缩得很快，可能很快满足条件，但也可能错过较好的较大步长。如果 (\beta) 接近 (1)，比如 (0.9)，搜索会更细，但可能需要更多次函数值计算。

第 9 页到这里实际上完成了前面“算法部分”的最后一块。我们已经知道了梯度下降的迭代格式、下降方向、负梯度方向、停止准则、常数步长、精确线搜索、Armijo 回溯线搜索。后面从第 10 页开始，讲义进入收敛分析。也就是说，不再只是描述算法怎么运行，而是要证明它为什么会下降、什么时候收敛、收敛速度是多少。

在进入第 10 页之前，你需要确认自己掌握 Armijo 条件的三层含义。第一，它要求新函数值比旧函数值低。第二，它不是随便低一点，而是要至少达到一阶预测下降量的一小部分。第三，回溯线搜索通过不断缩小步长来保证最终满足这个条件。

我们继续进入第 10 页。从这一页开始，讲义正式进入梯度下降法的收敛分析。前面我们已经理解了算法怎样走，现在要回答更理论的问题：它为什么会下降？它会不会收敛？如果收敛，速度有多快？

第 10 页考虑的是标准梯度下降迭代

[
x_{k+1}=x_k-t_k\nabla f(x_k).
]

讲义在这里列出三个基本假设。第一个假设是 (f) 可微，并且定义域是整个 (\mathbb{R}^n)。这表示我们在任何点都可以计算梯度，也不用担心一步走出去以后不在定义域里。第二个假设是 (\nabla f(x)) 是 Lipschitz 连续的，并且 Lipschitz 参数为 (L>0)。这个假设非常关键，后面所有下降量估计几乎都依赖它。第三个假设是最优值有下界，也就是说函数值不会无限下降到 (-\infty)。如果函数可以一直下降没有底，那么谈论收敛到某个稳定目标值就没有意义。

为了理解第 10 页，我们必须先认真讲第 11 页的 Lipschitz 连续。第 11 页定义说，一个函数 (h) 是 Lipschitz 连续的，如果存在常数 (L)，使得对所有 (x,y)，都有

[
|h(x)-h(y)|_2\le L|x-y|_2.
]

这句话的意思是：输入变化多少，输出变化不会超过它的 (L) 倍。也就是说，函数 (h) 的变化速度被一个统一常数 (L) 控制住了。它不能突然无限陡峭，不能在很小的输入变化下产生无限大的输出变化。

这里讲义真正关心的是 (h=\nabla f)，也就是梯度函数本身 Lipschitz 连续。于是条件变成

[
|\nabla f(x)-\nabla f(y)|_2\le L|x-y|_2.
]

这表示梯度变化不会太快。如果从 (x) 移动到 (y)，梯度可以变，但变化幅度最多和距离 (|x-y|) 成正比。这个性质通常叫 (L)-smooth，也就是 (L)-光滑。

为什么要控制梯度变化？因为梯度下降每一步使用的是当前点 (x_k) 的梯度信息。如果函数在附近变化太剧烈，那么当前梯度可能刚走出去一点就完全失效，算法很难分析。但如果梯度是 Lipschitz 连续的，我们就知道函数在局部不会“弯得太离谱”，于是可以用二次函数去控制它的上界。

第 11 页还写了记号 (C^{1,1}_L(\mathbb{R}^n))。这个记号的意思是：函数一阶可微，并且一阶导数 Lipschitz 连续，Lipschitz 常数为 (L)。第一个 (1) 表示有一阶导数，第二个 (1) 表示这个一阶导数满足 Lipschitz 条件。简单说，如果 (f\in C^{1,1}_L)，就是 (f) 可微且梯度 (L)-Lipschitz。

讲义给的例子包括线性函数和二次函数。比如

[
f(x)=a^Tx+b.
]

它的梯度是

[
\nabla f(x)=a.
]

这是一个常向量，所以对任意 (x,y)，

[
\nabla f(x)-\nabla f(y)=0.
]

因此它当然是 Lipschitz 连续的，甚至可以取 (L=0)。不过在优化分析中通常取某个非负或正的上界即可。

再看二次函数

[
f(x)=x^TAx+2b^Tx+c.
]

如果 (A) 是对称矩阵，那么

[
\nabla f(x)=2Ax+2b.
]

于是

[
\nabla f(x)-\nabla f(y)=2A(x-y).
]

由矩阵范数性质，

[
|\nabla f(x)-\nabla f(y)|
=========================

|2A(x-y)|
\le 2|A||x-y|.
]

所以它的梯度 Lipschitz 常数可以取

[
L=2|A|.
]

如果函数写成更常见的形式

[
f(x)=\frac12 x^TAx+b^Tx+c,
]

那么

[
\nabla f(x)=Ax+b,
]

对应 Lipschitz 常数可以取

[
L=|A|.
]

这也再次说明，不同教材中常数差异往往来自二次函数前面的系数写法。

接着进入第 12 页，它给出了 Lipschitz 梯度最重要的性质：二次上界。如果 (\nabla f) 是 Lipschitz 连续的，参数为 (L)，那么对任意 (x,y)，都有

[
f(y)\le f(x)+\nabla f(x)^T(y-x)+\frac{L}{2}|x-y|_2^2.
]

这个不等式是后面收敛分析的核心。它说，虽然 (f(y)) 可能不是线性函数，但它不会超过“在 (x) 处的一阶线性近似”再加上一个二次误差项。也就是说，函数的真实值可以被一个二次模型从上方控制住。

我们来推导这个不等式，因为它非常重要。令

[
d=y-x.
]

定义一维函数

[
\phi(s)=f(x+sd),\qquad s\in[0,1].
]

那么

[
\phi(0)=f(x),\qquad \phi(1)=f(y).
]

根据微积分基本定理，

[
f(y)-f(x)=\phi(1)-\phi(0)=\int_0^1 \phi'(s),ds.
]

而

[
\phi'(s)=\nabla f(x+sd)^Td.
]

所以

[
f(y)-f(x)=\int_0^1 \nabla f(x+sd)^Td,ds.
]

现在我们加减 (\nabla f(x)^Td)：

[
f(y)-f(x)
=========

\int_0^1 \left[\nabla f(x)^Td+\left(\nabla f(x+sd)-\nabla f(x)\right)^Td\right]ds.
]

第一部分积分为

[
\int_0^1 \nabla f(x)^Td,ds=\nabla f(x)^Td.
]

第二部分用 Cauchy-Schwarz 不等式估计：

[
\left(\nabla f(x+sd)-\nabla f(x)\right)^Td
\le
|\nabla f(x+sd)-\nabla f(x)||d|.
]

由于梯度 (L)-Lipschitz，

[
|\nabla f(x+sd)-\nabla f(x)|
\le
L|sd|
=====

Ls|d|.
]

所以

[
\left(\nabla f(x+sd)-\nabla f(x)\right)^Td
\le
Ls|d|^2.
]

积分得到

[
\int_0^1 Ls|d|^2,ds
===================

\frac{L}{2}|d|^2.
]

因此

[
f(y)-f(x)
\le
\nabla f(x)^Td+\frac{L}{2}|d|^2.
]

把 (d=y-x) 代回去，就得到

[
f(y)\le f(x)+\nabla f(x)^T(y-x)+\frac{L}{2}|y-x|^2.
]

这就是第 12 页的二次上界，也常叫 descent lemma 的基础形式。

这个不等式的直觉是：如果梯度变化被 (L) 控制，那么函数离开 (x) 后最多会比切线高出一个二次量。(L) 越大，说明函数可能弯得越厉害，上界越松；(L) 越小，说明函数越平滑，上界越紧。

第 12 页还说，这个性质等价于

[
\frac{L}{2}x^Tx-f(x)
]

是凸函数。这个结论可以先不作为主要工具，但我们可以理解它的含义。若 (f) 的曲率不超过 (L)，那么用一个曲率为 (L) 的二次函数减去 (f)，剩下的函数应该是凸的。换成二阶导数语言，如果 (f) 二阶可微，那么

[
\nabla^2\left(\frac{L}{2}x^Tx-f(x)\right)
=========================================

LI-\nabla^2 f(x).
]

这个函数凸等价于 Hessian 半正定，也就是

[
LI-\nabla^2 f(x)\succeq 0.
]

这等价于

[
\nabla^2 f(x)\preceq LI.
]

所以第 12 页写道，如果 (f) 二阶可微，梯度 Lipschitz 与 Hessian 有界有密切关系。更准确地说，若 (f) 在整个 (\mathbb{R}^n) 上二阶连续可微，则

[
f\in C^{1,1}_L(\mathbb{R}^n)
]

当且仅当

[
|\nabla^2f(x)|\le L
]

对所有 (x\in\mathbb{R}^n) 成立。

这句话非常有用。它告诉我们，梯度 Lipschitz 本质上是在限制 Hessian 的大小，也就是限制函数的最大曲率。对于一维函数来说，Hessian 就是二阶导数，所以条件变成

[
|f''(x)|\le L.
]

也就是说，二阶导数不能无限大。

第 12 页最后给了例子

[
f(x)=\sqrt{1+x^2}.
]

我们可以看一下它为什么是光滑的。一维情况下，

[
f'(x)=\frac{x}{\sqrt{1+x^2}}.
]

继续求导：

[
f''(x)=\frac{1}{(1+x^2)^{3/2}}.
]

显然

[
0<f''(x)\le 1.
]

所以可以取

[
L=1.
]

这说明 (\nabla f=f') 是 Lipschitz 连续的，Lipschitz 常数可以取 (1)。

现在把第 10 到第 12 页串起来。第 10 页说我们要分析梯度下降，需要假设 (f) 可微、梯度 Lipschitz、函数有下界。第 11 页定义了什么叫梯度 Lipschitz。第 12 页说明梯度 Lipschitz 带来的关键工具是二次上界：

[
f(y)\le f(x)+\nabla f(x)^T(y-x)+\frac{L}{2}|y-x|^2.
]

下一页第 13 页会把这个不等式代入梯度下降更新

[
y=x-t\nabla f(x)
]

中，从而推出充分下降引理：

[
f(x)-f(x-t\nabla f(x))
\ge
t\left(1-\frac{Lt}{2}\right)|\nabla f(x)|^2.
]

这将是证明梯度下降函数值下降和梯度趋于零的关键一步。

在继续之前，你需要确保自己理解这一段的主线：Lipschitz 梯度不是一个孤立定义，它的真正作用是给出二次上界；二次上界告诉我们函数不会比线性近似高太多；把梯度下降的一步代入这个二次上界，就能得到每一步至少下降多少。

好，我们现在专门讲第 13 页的充分下降引理。它是梯度下降收敛分析里的第一个关键结论。前面我们已经知道，如果 (\nabla f) 是 (L)-Lipschitz 连续的，那么对任意 (x,y)，有二次上界

[
f(y)\le f(x)+\nabla f(x)^T(y-x)+\frac{L}{2}|y-x|^2.
]

第 13 页就是把这个不等式用于梯度下降的一步。梯度下降从 (x) 出发，沿负梯度方向走一步：

[
y=x-t\nabla f(x),
]

其中 (t>0) 是步长。我们要估计的是：从 (x) 到 (x-t\nabla f(x)) 之后，函数值至少下降多少。讲义给出的结论是

[
f(x)-f(x-t\nabla f(x))
\ge
t\left(1-\frac{Lt}{2}\right)|\nabla f(x)|^2.
]

这就是充分下降引理。

我们一步一步推导。由二次上界，取

[
y=x-t\nabla f(x).
]

于是

[
y-x=-t\nabla f(x).
]

把它代入二次上界：

[
f(x-t\nabla f(x))
\le
f(x)+\nabla f(x)^T(-t\nabla f(x))
+\frac{L}{2}|-t\nabla f(x)|^2.
]

现在分别化简第二项和第三项。第二项是

[
\nabla f(x)^T(-t\nabla f(x))
============================

-t|\nabla f(x)|^2.
]

第三项是

[
\frac{L}{2}|-t\nabla f(x)|^2
============================

\frac{L}{2}t^2|\nabla f(x)|^2.
]

所以

[
f(x-t\nabla f(x))
\le
f(x)-t|\nabla f(x)|^2+\frac{L}{2}t^2|\nabla f(x)|^2.
]

把后两项合并：

[
f(x-t\nabla f(x))
\le
f(x)-t\left(1-\frac{Lt}{2}\right)|\nabla f(x)|^2.
]

现在把右边的下降项移到左边，就得到

[
f(x)-f(x-t\nabla f(x))
\ge
t\left(1-\frac{Lt}{2}\right)|\nabla f(x)|^2.
]

这就是讲义第 13 页的充分下降引理。

这个不等式的意义非常重要。左边

[
f(x)-f(x-t\nabla f(x))
]

是真实函数值下降量。右边是一个可以明确计算的下界。它告诉我们，只要步长 (t) 选得合适，每一步下降量至少和梯度范数平方成正比。

关键在于系数

[
t\left(1-\frac{Lt}{2}\right).
]

如果这个系数是正的，那么只要 (\nabla f(x)\ne 0)，右边就严格大于零，于是函数值确实下降。这个系数什么时候为正？因为 (t>0)，所以需要

[
1-\frac{Lt}{2}>0.
]

也就是

[
t<\frac{2}{L}.
]

因此充分下降引理告诉我们：在 (L)-smooth 条件下，如果梯度下降的步长满足

[
0<t<\frac{2}{L},
]

那么每一步都能保证函数值下降。特别常见的选择是

[
t=\frac{1}{L}.
]

把它代入右边的系数：

[
t\left(1-\frac{Lt}{2}\right)
============================

# \frac1L\left(1-\frac12\right)

\frac{1}{2L}.
]

于是得到

[
f(x)-f\left(x-\frac1L\nabla f(x)\right)
\ge
\frac{1}{2L}|\nabla f(x)|^2.
]

这说明当步长取 (1/L) 时，每一步至少下降

[
\frac{1}{2L}|\nabla f(x)|^2.
]

这个式子以后会反复出现。

我们也可以从几何上理解这个引理。二次上界说，真实函数不会超过一个局部二次模型：

[
f(x)+\nabla f(x)^T(y-x)+\frac{L}{2}|y-x|^2.
]

梯度下降选择 (y=x-t\nabla f(x))。其中线性项

[
-t|\nabla f(x)|^2
]

带来下降，而二次项

[
\frac{L}{2}t^2|\nabla f(x)|^2
]

抵消一部分下降，因为函数可能弯曲，真实函数值不一定像线性预测那样下降得那么多。最终净下降至少是

[
t|\nabla f(x)|^2-\frac{L}{2}t^2|\nabla f(x)|^2
==============================================

t\left(1-\frac{Lt}{2}\right)|\nabla f(x)|^2.
]

所以这个引理本质上是在说：梯度下降的下降量等于“线性下降收益”减去“曲率带来的损失”。步长太小时，线性下降收益很小；步长太大时，曲率损失可能超过收益。步长适中时，才能保证下降。

这里有一个容易忽略但很重要的点。充分下降引理本身并不要求 (f) 是凸函数。它只要求 (\nabla f) 是 Lipschitz 连续，并且函数有下界用于后续收敛分析。也就是说，即使 (f) 是非凸函数，只要它是 (L)-smooth，梯度下降仍然可以保证在合适步长下函数值下降。只是对于非凸函数，我们后面只能证明梯度趋于零，不能保证找到全局最优点。

第 13 页还假设 (f) 在 (\mathbb{R}^n) 上有下界，即存在某个 (m)，使得

[
f(x)>m
]

对所有 (x) 成立。这个假设在充分下降引理本身的推导中没有用到，但在后面证明收敛时非常重要。因为如果每一步都让函数值下降，而函数值又有下界，那么函数值序列

[
f(x_0),f(x_1),f(x_2),\ldots
]

就不会无限下降，它会收敛到某个有限值。这个事实会进一步帮助我们证明

[
\nabla f(x_k)\to 0.
]

现在我们继续把第 14 页也顺着讲掉，因为它是在第 13 页充分下降引理基础上，分别讨论常数步长、精确线搜索和回溯线搜索下能得到什么样的下降量。

对于常数步长，假设每一步都取同一个

[

根据充分下降引理，直接有

[

只要

括号里的系数就是正的。讲义中特别说，常用选择是

[

这时下降量下界就是

[
f(x)-f\left(x-\frac1L\nabla f(x)\right)
\ge
\frac{1}{2L}|\nabla f(x)|^2.

对于精确线搜索，讲义写道：

[
f(x)-f(x-\bar t\nabla f(x))

这里的意思是，精确线搜索选择的是沿负梯度方向让函数值最小的步长。既然步长 (1/L) 是一个可选步长，那么精确线搜索得到的函数值不可能比使用 (1/L) 更差。也就是说，设

所以

]

这说明精确线搜索至少不差于固定步长 (1/L) 所能保证的下降量。

对于回溯线搜索，第 14 页给出了一个稍复杂的下降量：

这个式子乍看复杂，但它的意思很清楚：Armijo 回溯线搜索虽然不精确求最优步长，但仍然可以保证每一步下降量至少和

我们解释一下这个常数为什么是这样的。Armijo 条件对梯度下降方向 (p=-\nabla f(x)) 变成

左边是一个望远镜求和：

[
\sum_{i=0}^k \left(f(x_i)-f(x_{i+1})\right)
===========================================

f(x_0)-f(x_{k+1}).
]

所以

[
f(x_0)-f(x_{k+1})
\ge
M\sum_{i=0}^k |\nabla f(x_i)|^2.
]

也就是

[
\sum_{i=0}^k |\nabla f(x_i)|^2
\le
\frac{f(x_0)-f(x_{k+1})}{M}.
]

令 (k\to\infty)，由于 (f(x_{k+1})\to f^\ast)，得到

[
\sum_{i=0}^{\infty} |\nabla f(x_i)|^2
\le
\frac{f(x_0)-f^\ast}{M}
<\infty.
]

这说明非负级数

[
\sum_{i=0}^{\infty} |\nabla f(x_i)|^2
]

收敛。因此它的通项必须趋于零：

[
|\nabla f(x_k)|^2\to 0.
]

于是

[
|\nabla f(x_k)|\to 0.
]

这就证明了梯度趋于零。

再讲第 15 页的第二个结论：梯度范数的收敛速度。讲义写的是，对于任意

[
k=0,1,2,\ldots,
]

有

[
\min_{i=0,1,\ldots,k}|\nabla f(x_i)|
\le
\sqrt{
\frac{f(x_0)-f^\ast}{(k+1)M}
}.
]

这个式子看起来有点长，但它的含义很简单：在前 (k+1) 次迭代中，至少有一次的梯度范数不超过右边。而右边随着 (k) 增大大约像

[
\frac{1}{\sqrt{k}}
]

下降，所以这叫 (O(1/\sqrt{k})) 的梯度范数收敛速度。

我们来推导。前面已经得到

[
M\sum_{i=0}^k|\nabla f(x_i)|^2
\le
f(x_0)-f(x_{k+1})
\le
f(x_0)-f^\ast.
]

所以

[
\sum_{i=0}^k|\nabla f(x_i)|^2
\le
\frac{f(x_0)-f^\ast}{M}.
]

现在注意，在 (k+1) 个非负数

[
|\nabla f(x_0)|^2,\ldots,|\nabla f(x_k)|^2
]

中，最小值一定不超过平均值。因此

[
\min_{i=0,\ldots,k}|\nabla f(x_i)|^2
\le
\frac{1}{k+1}
\sum_{i=0}^k|\nabla f(x_i)|^2.
]

再代入上面的和式上界，得到

[
\min_{i=0,\ldots,k}|\nabla f(x_i)|^2
\le
\frac{f(x_0)-f^\ast}{(k+1)M}.
]

两边开平方：

[
\min_{i=0,\ldots,k}|\nabla f(x_i)|
\le
\sqrt{
\frac{f(x_0)-f^\ast}{(k+1)M}
}.
]

这就是讲义第 15 页的结论。

为什么这里是

[
\min_{i=0,\ldots,k}
]

而不是直接写

[
|\nabla f(x_k)|?
]

这是因为在一般非凸情况下，我们很容易控制“前 (k+1) 次中最好的一次”，但不一定能简单地证明每一步的梯度范数都按同样速度下降。函数值是单调下降的，但梯度范数本身可能会波动。因此我们通常给出这种形式的结果：在前 (k) 步中，至少有一个点的梯度已经足够小。

这个结果在非凸优化里非常常见。它告诉我们，如果想找到一个满足

[
|\nabla f(x_i)|\le \epsilon
]

的点，那么只需要让右边小于等于 (\epsilon)。也就是要求

[
\sqrt{
\frac{f(x_0)-f^\ast}{(k+1)M}
}
\le \epsilon.
]

两边平方：

[
\frac{f(x_0)-f^\ast}{(k+1)M}
\le
\epsilon^2.
]

所以

[
k+1
\ge
\frac{f(x_0)-f^\ast}{M\epsilon^2}.
]

这说明为了找到梯度范数不超过 (\epsilon) 的点，迭代次数大约是

[
O\left(\frac{1}{\epsilon^2}\right).
]

这和刚才的 (O(1/\sqrt{k})) 是同一件事的两种说法。

最后看第 15 页的 remark。讲义强调，这个

[
O(1/\sqrt{k})
]

的速率确保收敛到一个 stationary point，也就是驻点，比如局部最小点，但不保证找到全局最优。原因是我们目前没有假设 (f) 是凸函数。对于非凸函数，满足

[
\nabla f(x)=0
]

的点可能有很多，而且算法最后靠近哪一个，很大程度上取决于初始点和函数形状。

我们把第 15 页总结成一条主线。由于充分下降引理，我们知道每一步下降量至少为

[
M|\nabla f(x_k)|^2.
]

由于函数值有下界，总下降量不可能无限大。因此所有梯度范数平方的总和有限。既然总和有限，梯度范数必须趋于零。这就是一般光滑函数下梯度下降的基本收敛性。进一步，用“最小值不超过平均值”可以得到前 (k+1) 次迭代中最小梯度范数的 (O(1/\sqrt{k})) 速率。

你现在不需要记住所有细节，但要抓住这条证明链：

[
\text{充分下降}
\Rightarrow
\text{函数值单调下降}
\Rightarrow
\text{总下降量有限}
\Rightarrow
\sum |\nabla f(x_k)|^2<\infty
\Rightarrow
|\nabla f(x_k)|\to 0.
]

如果你能理解这条链，第 15 页就掌握了。

好，我们继续第 16 页。前面第 13 到第 15 页分析的是“一般光滑函数”，也就是说不要求凸。在那种情况下，我们只能证明梯度范数趋于零，也就是逐渐接近驻点。现在第 16 页开始加入更强的结构：假设 (f) 是凸函数。只要有凸性，我们就可以证明函数值真的趋近全局最优值，而且能得到更明确的函数值误差收敛速度。

这一页考虑的是常数步长梯度下降：

[
x_{k+1}=x_k-t\nabla f(x_k).
]

这里 (t) 是固定步长。为了得到讲义中的结论，通常还隐含要求 (f) 是凸函数、(\nabla f) 是 (L)-Lipschitz 连续，并且步长选得合适，例如 (0<t\le 1/L)。讲义给出的核心结论是

[
f(x_k)-f(x^\ast)\le \frac{1}{2kt}|x_0-x^\ast|_2^2.
]

这个式子非常重要。它说明凸函数情形下，梯度下降的函数值误差按照

[
O\left(\frac1k\right)
]

的速度下降。这里 (x^\ast) 是全局最优点，(f(x^\ast)) 是全局最优值。

我们现在来推导这个结论。推导的核心思想是：看每一步之后当前点到最优点的距离平方如何变化。因为

[
x_{k+1}=x_k-t\nabla f(x_k),
]

所以

[
x_{k+1}-x^\ast=x_k-x^\ast-t\nabla f(x_k).
]

两边取范数平方：

[
|x_{k+1}-x^\ast|^2
==================

|x_k-x^\ast-t\nabla f(x_k)|^2.
]

展开平方：

[
|x_{k+1}-x^\ast|^2
==================

|x_k-x^\ast|^2
-2t\nabla f(x_k)^T(x_k-x^\ast)
+t^2|\nabla f(x_k)|^2.
]

现在我们需要把右边和函数值误差 (f(x_k)-f(x^\ast)) 联系起来。首先，由凸函数的一阶性质，对任意 (x,y)，有

[
f(y)\ge f(x)+\nabla f(x)^T(y-x).
]

取

[
x=x_k,\qquad y=x^\ast,
]

得到

[
f(x^\ast)\ge f(x_k)+\nabla f(x_k)^T(x^\ast-x_k).
]

移项：

[
\nabla f(x_k)^T(x_k-x^\ast)\ge f(x_k)-f(x^\ast).
]

这一步非常关键。它说明当前梯度和“从最优点指向当前点的方向”的内积，至少等于当前函数值误差。

接下来我们还要处理最后一项

[
t^2|\nabla f(x_k)|^2.
]

由于 (f) 是 (L)-smooth，并且步长 (t\le 1/L)，可以利用一个常见结论：

[
\frac{t}{2}|\nabla f(x_k)|^2\le f(x_k)-f(x_{k+1}).
]

这个结论其实来自我们前面讲过的充分下降引理。当 (0<t\le 1/L) 时，

[
f(x_k)-f(x_{k+1})
\ge
t\left(1-\frac{Lt}{2}\right)|\nabla f(x_k)|^2.
]

因为 (t\le 1/L)，所以

[
1-\frac{Lt}{2}\ge \frac12.
]

于是

[
f(x_k)-f(x_{k+1})
\ge
\frac{t}{2}|\nabla f(x_k)|^2.
]

等价地，

[
t^2|\nabla f(x_k)|^2
\le
2t\bigl(f(x_k)-f(x_{k+1})\bigr).
]

把这些代入距离平方展开式：

[
|x_{k+1}-x^\ast|^2
==================

|x_k-x^\ast|^2
-2t\nabla f(x_k)^T(x_k-x^\ast)
+t^2|\nabla f(x_k)|^2.
]

由凸性，

[
\nabla f(x_k)^T(x_k-x^\ast)\ge f(x_k)-f(x^\ast),
]

所以

[
-2t\nabla f(x_k)^T(x_k-x^\ast)
\le
-2t(f(x_k)-f(x^\ast)).
]

再由充分下降，

[
t^2|\nabla f(x_k)|^2
\le
2t(f(x_k)-f(x_{k+1})).
]

因此

[
|x_{k+1}-x^\ast|^2
\le
|x_k-x^\ast|^2
-2t(f(x_k)-f(x^\ast))
+2t(f(x_k)-f(x_{k+1})).
]

中间两项合并：

[
-2t(f(x_k)-f(x^\ast))
+2t(f(x_k)-f(x_{k+1}))
======================

2t(f(x^\ast)-f(x_{k+1})).
]

所以

[
|x_{k+1}-x^\ast|^2
\le
|x_k-x^\ast|^2
-2t(f(x_{k+1})-f(x^\ast)).
]

移项得到

[
2t(f(x_{k+1})-f(x^\ast))
\le
|x_k-x^\ast|^2-|x_{k+1}-x^\ast|^2.
]

这就是推导的核心不等式。它的含义是：第 (k+1) 步的函数值误差，可以被“距离最优点的平方减少量”控制。

现在对 (k=0,1,\ldots,N-1) 求和：

[
2t\sum_{k=0}^{N-1}(f(x_{k+1})-f(x^\ast))
\le
\sum_{k=0}^{N-1}
\left(
|x_k-x^\ast|^2-|x_{k+1}-x^\ast|^2
\right).
]

右边又是望远镜求和：

[
\sum_{k=0}^{N-1}
\left(
|x_k-x^\ast|^2-|x_{k+1}-x^\ast|^2
\right)
=======

|x_0-x^\ast|^2-|x_N-x^\ast|^2.
]

因为范数平方非负，所以

[
|x_0-x^\ast|^2-|x_N-x^\ast|^2
\le
|x_0-x^\ast|^2.
]

于是

[
2t\sum_{k=0}^{N-1}(f(x_{k+1})-f(x^\ast))
\le
|x_0-x^\ast|^2.
]

也就是

[
\sum_{k=1}^{N}(f(x_k)-f(x^\ast))
\le
\frac{1}{2t}|x_0-x^\ast|^2.
]

因为梯度下降在合适步长下函数值非增，所以

[
f(x_N)\le f(x_k)
]

对 (k=1,\ldots,N) 成立，因此

[
f(x_N)-f(x^\ast)
\le
\frac1N\sum_{k=1}^{N}(f(x_k)-f(x^\ast)).
]

把上面的和式估计代入，就得到

[
f(x_N)-f(x^\ast)
\le
\frac{1}{2Nt}|x_0-x^\ast|^2.
]

把 (N) 改写成 (k)，就是讲义中的结论：

[
f(x_k)-f(x^\ast)\le \frac{1}{2kt}|x_0-x^\ast|^2.
]

这个结论的意义是：如果我们想要

[
f(x_k)-f(x^\ast)\le \epsilon,
]

只需要让

[
\frac{1}{2kt}|x_0-x^\ast|^2\le \epsilon.
]

解出 (k)：

[
k\ge \frac{|x_0-x^\ast|^2}{2t\epsilon}.
]

所以迭代次数随 (1/\epsilon) 增长，记作

[
O\left(\frac1\epsilon\right).
]

这就是第 16 页中 “iterate number for (f(x_k)-f(x^\ast)\le \epsilon) is (O(1/\epsilon))” 的意思。注意这里说的是为了达到函数值误差 (\epsilon)，需要大约 (1/\epsilon) 量级的迭代次数。

第 16 页还给出一个梯度范数的估计：

[
|\nabla f(x_k)|^2
\le
2L(f(x_k)-f^\ast)
\le
\frac{L}{kt}|x_0-x^\ast|_2^2.
]

这里的第一个不等式也值得解释。对于凸且 (L)-smooth 的函数，有一个常用关系：

[
|\nabla f(x)|^2\le 2L(f(x)-f^\ast).
]

它可以这样理解。由充分下降引理，取步长 (1/L)，有

[
f(x)-f\left(x-\frac1L\nabla f(x)\right)
\ge
\frac{1}{2L}|\nabla f(x)|^2.
]

因为 (f^\ast) 是全局最小值，所以

[
f\left(x-\frac1L\nabla f(x)\right)\ge f^\ast.
]

于是

[
f(x)-f^\ast
\ge
f(x)-f\left(x-\frac1L\nabla f(x)\right)
\ge
\frac{1}{2L}|\nabla f(x)|^2.
]

移项得到

[
|\nabla f(x)|^2\le 2L(f(x)-f^\ast).
]

再把刚才的函数值误差界代入：

[
f(x_k)-f^\ast
\le
\frac{1}{2kt}|x_0-x^\ast|^2,
]

于是

[
|\nabla f(x_k)|^2
\le
2L\cdot \frac{1}{2kt}|x_0-x^\ast|^2
===================================

\frac{L}{kt}|x_0-x^\ast|^2.
]

这就得到讲义中的第二个结论。

这一页最后提到 Nesterov 的快速梯度方法，它可以把凸光滑优化中的函数值收敛速度从普通梯度下降的

[
O\left(\frac1k\right)
]

提升到

[
O\left(\frac1{k^2}\right).
]

这里我们暂时不展开，因为这份讲义的主体还是经典梯度下降。你现在只需要知道：普通梯度下降在一般凸光滑情形下是 (O(1/k))，而 Nesterov 加速方法可以达到 (O(1/k^2))，这是凸优化中非常重要的加速结果。

我们把第 16 页总结一下。一般非凸情形下，梯度下降只能保证梯度趋于零；一旦加入凸性，就可以保证函数值趋于全局最优值。对于常数步长梯度下降，在合适步长下，有

[
f(x_k)-f(x^\ast)\le \frac{1}{2kt}|x_0-x^\ast|^2.
]

这说明函数值误差是 (O(1/k))，达到误差 (\epsilon) 需要 (O(1/\epsilon)) 次迭代。

这一页最重要的证明逻辑是：利用凸性把

[
\nabla f(x_k)^T(x_k-x^\ast)
]

和函数值误差联系起来，再利用充分下降控制梯度范数项，最后通过距离平方的望远镜求和得到 (O(1/k)) 速度。

好，我们继续第 17 页。第 16 页讲的是凸函数：有了凸性以后，梯度下降可以保证函数值收敛到全局最优值，并且普通梯度下降的函数值误差是 (O(1/k))。第 17 页进一步加强假设，从“凸”升级到“强凸”。一旦函数强凸，梯度下降的收敛速度会从次线性 (O(1/k)) 提升为线性收敛，也就是误差每一步大约乘以一个固定小于 1 的比例。

先看强凸的定义。讲义写道，(f) 是 (\mu)-strongly convex，如果

[
f(x)-\frac{\mu}{2}x^Tx
]

是凸函数，其中 (\mu>0)。这个定义的含义是：(f) 不只是凸的，而且至少有一个二次函数那么“弯”。普通凸函数可能很平，比如 (f(x)=x^4) 在 (x=0) 附近非常平；强凸函数则要求整体上至少有 (\mu) 这么大的曲率，因此它不会出现无限平坦的谷底。

如果用几何直觉理解，普通凸函数像一个碗，但碗底可能很平；强凸函数像一个有明确弯曲程度的碗，不能太平。参数 (\mu) 越大，说明函数下方弯曲越强，最优点附近越“尖”一些；(\mu) 越小，说明函数可能更扁，优化更慢。

强凸有一个非常重要的一阶刻画，也就是讲义写的二次下界：

[
f(y)\ge f(x)+\nabla f(x)^T(y-x)+\frac{\mu}{2}|x-y|_2^2.
]

这个式子和第 12 页的光滑性二次上界正好对应。光滑性给的是上界：

[
f(y)\le f(x)+\nabla f(x)^T(y-x)+\frac{L}{2}|x-y|^2.
]

强凸性给的是下界：

[
f(y)\ge f(x)+\nabla f(x)^T(y-x)+\frac{\mu}{2}|x-y|^2.
]

所以如果一个函数既是 (L)-smooth，又是 (\mu)-strongly convex，那么它被夹在两个二次模型之间。它的曲率不会超过 (L)，也不会低于 (\mu)。这就是后面条件数 (L/\mu) 的来源。

我们稍微解释强凸二次下界为什么成立。因为

[
g(x)=f(x)-\frac{\mu}{2}|x|^2
]

是凸函数，所以由凸函数的一阶性质，有

[
g(y)\ge g(x)+\nabla g(x)^T(y-x).
]

而

[
\nabla g(x)=\nabla f(x)-\mu x.
]

代入得到

[
f(y)-\frac{\mu}{2}|y|^2
\ge
f(x)-\frac{\mu}{2}|x|^2+
(\nabla f(x)-\mu x)^T(y-x).
]

把右边展开：

[
f(y)\ge f(x)+\nabla f(x)^T(y-x)
+\frac{\mu}{2}|y|^2-\frac{\mu}{2}|x|^2-\mu x^T(y-x).
]

现在化简后面三个关于 (\mu) 的项。注意

[
-\mu x^T(y-x)=-\mu x^Ty+\mu x^Tx.
]

因此

[
\frac{\mu}{2}|y|^2-\frac{\mu}{2}|x|^2-\mu x^T(y-x)
==================================================

\frac{\mu}{2}|y|^2+\frac{\mu}{2}|x|^2-\mu x^Ty.
]

而

[
|y-x|^2=|y|^2+|x|^2-2x^Ty.
]

所以

[
\frac{\mu}{2}|y|^2+\frac{\mu}{2}|x|^2-\mu x^Ty
==============================================

\frac{\mu}{2}|y-x|^2.
]

于是得到

[
f(y)\ge f(x)+\nabla f(x)^T(y-x)+\frac{\mu}{2}|y-x|^2.
]

这就是强凸的一阶二次下界。

如果 (f) 二阶可微，那么强凸也可以用 Hessian 表示。讲义写的是

[
\nabla^2 f(x)\succeq \mu I.
]

这表示对任意方向 (d)，都有

[
d^T\nabla^2 f(x)d\ge \mu|d|^2.
]

也就是说，函数在任意方向上的曲率至少是 (\mu)。这和 (L)-smooth 的二阶条件

[
\nabla^2 f(x)\preceq LI
]

正好配对。如果两者同时成立，就有

[
\mu I\preceq \nabla^2 f(x)\preceq LI.
]

对于二次函数

[
f(x)=\frac12x^TAx+b^Tx+c,
]

Hessian 就是

[
\nabla^2 f(x)=A.
]

如果 (A) 正定，并且它的最小特征值是 (\lambda_{\min}(A))，最大特征值是 (\lambda_{\max}(A))，那么可以取

[
\mu=\lambda_{\min}(A),\qquad L=\lambda_{\max}(A).
]

这时强凸参数就是最小曲率，光滑参数就是最大曲率。

现在看第 17 页最重要的收敛结论：

[
f(x_k)-f^\ast
\le
\left(1-\frac{\mu}{L}\right)^k
\bigl(f(x_0)-f^\ast\bigr).
]

这里默认使用的是常数步长

[
t=\frac1L.
]

这个结论叫线性收敛。它不是说误差像一条直线下降，而是说每一步误差都会乘以一个固定比例

[
1-\frac{\mu}{L}.
]

由于

[
0<\mu\le L,
]

所以

[
0\le 1-\frac{\mu}{L}<1.
]

于是误差按几何级数衰减。

我们来推导这个结论。由 (L)-smooth 的充分下降结果，当取步长 (1/L) 时，有

[
f(x_{k+1})
\le
f(x_k)-\frac{1}{2L}|\nabla f(x_k)|^2.
]

如果我们能把

[
|\nabla f(x_k)|^2
]

用函数值误差

[
f(x_k)-f^\ast
]

从下方控制，就能得到函数值误差的递推。

强凸函数有一个常用不等式：

[
|\nabla f(x)|^2\ge 2\mu(f(x)-f^\ast).
]

这个不等式和第 16 页中凸光滑函数的不等式

[
|\nabla f(x)|^2\le 2L(f(x)-f^\ast)
]

方向相反。第 16 页的式子说，函数值误差大时梯度不能太大；这里的强凸式子说，函数值误差大时梯度不能太小。强凸性排除了“离最优值还很远但梯度已经很小”的情况。

我们证明一下这个强凸不等式。由强凸二次下界，对任意 (y)，

[
f(y)\ge f(x)+\nabla f(x)^T(y-x)+\frac{\mu}{2}|y-x|^2.
]

右边是关于 (y) 的二次函数。因为这个不等式对所有 (y) 成立，所以 (f^\ast=\min_y f(y)) 也满足

[
f^\ast\ge
\min_y
\left[
f(x)+\nabla f(x)^T(y-x)+\frac{\mu}{2}|y-x|^2
\right].
]

令

[
s=y-x.
]

则要最小化

[
f(x)+\nabla f(x)^Ts+\frac{\mu}{2}|s|^2.
]

这是一个简单二次函数，对 (s) 求导：

[
\nabla f(x)+\mu s=0.
]

所以

[
s=-\frac1\mu\nabla f(x).
]

代入最小值：

[
\nabla f(x)^Ts+\frac{\mu}{2}|s|^2
=================================

-\frac1\mu|\nabla f(x)|^2
+\frac{\mu}{2}\cdot\frac1{\mu^2}|\nabla f(x)|^2
===============================================

-\frac{1}{2\mu}|\nabla f(x)|^2.
]

所以

[
f^\ast
\ge
f(x)-\frac{1}{2\mu}|\nabla f(x)|^2.
]

移项得到

[
f(x)-f^\ast
\le
\frac{1}{2\mu}|\nabla f(x)|^2.
]

也就是

[
|\nabla f(x)|^2
\ge
2\mu(f(x)-f^\ast).
]

现在把它代入充分下降式：

[
f(x_{k+1})
\le
f(x_k)-\frac{1}{2L}|\nabla f(x_k)|^2.
]

由

[
|\nabla f(x_k)|^2
\ge
2\mu(f(x_k)-f^\ast),
]

得到

[
f(x_{k+1})
\le
f(x_k)-\frac{1}{2L}\cdot 2\mu(f(x_k)-f^\ast).
]

也就是

[
f(x_{k+1})
\le
f(x_k)-\frac{\mu}{L}(f(x_k)-f^\ast).
]

两边减去 (f^\ast)：

[
f(x_{k+1})-f^\ast
\le
f(x_k)-f^\ast-\frac{\mu}{L}(f(x_k)-f^\ast).
]

合并：

[
f(x_{k+1})-f^\ast
\le
\left(1-\frac{\mu}{L}\right)(f(x_k)-f^\ast).
]

这就是一步收缩关系。反复迭代得到

[
f(x_k)-f^\ast
\le
\left(1-\frac{\mu}{L}\right)^k(f(x_0)-f^\ast).
]

这正是讲义第 17 页的线性收敛结论。

现在我们比较一下第 16 页和第 17 页。对于一般凸光滑函数，误差界是

[
f(x_k)-f^\ast\le \frac{1}{2kt}|x_0-x^\ast|^2,
]

这是 (O(1/k))。如果要把误差降到 (\epsilon)，迭代次数是 (O(1/\epsilon))。而强凸光滑情形下，误差界是

[
f(x_k)-f^\ast
\le
\left(1-\frac{\mu}{L}\right)^k(f(x_0)-f^\ast).
]

这是几何下降。如果要达到 (\epsilon)，只需要对数级别的精度依赖，也就是大致 (O(\log(1/\epsilon)))，不过前面还会乘上与 (L/\mu) 有关的因子。这个因子就是第 18 页要讲的条件数。

第 17 页最后还说，这个结果可以扩展到线搜索情形，如果假设 Armijo 参数 (\alpha=1/2)。这句话的意思是，不一定非要固定步长 (1/L)，某些线搜索方法也能给出类似的线性收敛结论，只是常数可能不同。因为线性收敛证明的核心仍然是两件事：每一步有足够下降，以及强凸性保证梯度范数能控制函数值误差。

这一页你最需要掌握的是三组对应关系。第一，光滑性是上界：

[
f(y)\le f(x)+\nabla f(x)^T(y-x)+\frac{L}{2}|y-x|^2.
]

第二，强凸性是下界：

[
f(y)\ge f(x)+\nabla f(x)^T(y-x)+\frac{\mu}{2}|y-x|^2.
]

第三，两者合起来给出线性收敛：

[
f(x_k)-f^\ast
\le
\left(1-\frac{\mu}{L}\right)^k(f(x_0)-f^\ast).
]

直觉上，(L) 控制“最陡的地方有多陡”，(\mu) 控制“最平的地方至少有多弯”。如果 (\mu) 和 (L) 接近，函数像圆碗，收敛快；如果 (\mu) 很小而 (L) 很大，函数像狭长山谷，收敛慢。下一页第 18 页就会把这个比例正式定义为条件数。

我们继续第 18 页，主题是 condition number，也就是条件数。上一页我们得到强凸光滑情形下梯度下降的线性收敛率：

[
f(x_k)-f^\ast
\le
\left(1-\frac{\mu}{L}\right)^k
\bigl(f(x_0)-f^\ast\bigr).
]

第 18 页就是在解释这个式子背后的速度因素。它告诉我们，梯度下降快不快，主要取决于

[
\frac{L}{\mu}.
]

这个比值就叫条件数。

先从矩阵条件数说起。对于一个正定矩阵 (A)，它的条件数定义为

[
\kappa(A)=\frac{\lambda_{\max}(A)}{\lambda_{\min}(A)}.
]

这里 (\lambda_{\max}(A)) 是最大特征值，(\lambda_{\min}(A)) 是最小特征值。因为 (A) 正定，所以所有特征值都大于零，因此这个比值有意义。条件数越接近 1，说明矩阵在各个方向上的伸缩比较均匀；条件数越大，说明不同方向上的伸缩差异越大。

对于优化问题，我们可以把这个概念推广到函数。若 (f) 是 (L)-smooth 且 (\mu)-strongly convex，那么它的曲率满足

[
\mu I\preceq \nabla^2 f(x)\preceq LI.
]

这表示任意方向上的曲率至少是 (\mu)，至多是 (L)。于是

[
\frac{L}{\mu}
]

就可以看成函数曲率的最大值与最小值之比，也就是函数的条件数。讲义中说 (L/\mu) is known as the condition number，意思就是这个。对于二次函数

[
f(x)=\frac12x^TAx+b^Tx+c,
]

如果 (A\succ0)，那么

[
\nabla^2 f(x)=A.
]

此时可以取

[
L=\lambda_{\max}(A),\qquad \mu=\lambda_{\min}(A),
]

所以函数的条件数就是

[
\frac{L}{\mu}
=============

# \frac{\lambda_{\max}(A)}{\lambda_{\min}(A)}

\kappa(A).
]

也就是说，二次函数的优化难度和 Hessian 矩阵的条件数直接相关。

现在看第 18 页中的迭代次数估计。我们希望达到

[
f(x_k)-f^\ast\le \epsilon.
]

由线性收敛界，

[
f(x_k)-f^\ast
\le
\left(1-\frac{\mu}{L}\right)^k
(f(x_0)-f^\ast).
]

为了让右边不超过 (\epsilon)，需要

[
\left(1-\frac{\mu}{L}\right)^k
(f(x_0)-f^\ast)
\le
\epsilon.
]

两边除以初始误差：

[
\left(1-\frac{\mu}{L}\right)^k
\le
\frac{\epsilon}{f(x_0)-f^\ast}.
]

取对数。因为 (0<1-\mu/L<1)，它的对数是负数，所以处理时要小心。等价写法是

[
k
\ge
\frac{
\log\left(\frac{f(x_0)-f^\ast}{\epsilon}\right)
}{
\log\left((1-\mu/L)^{-1}\right)
}.
]

这正是讲义中给出的形式：

[
\frac{\log((f(x_0)-f^\ast)/\epsilon)}
{\log((1-\mu/L)^{-1})}.
]

这个式子虽然精确，但不够直观。为了看出主要因素，我们用近似公式。若 (a) 很小，则

[
-\log(1-a)\approx a.
]

这里

[
a=\frac{\mu}{L}.
]

于是

[
\log\left((1-\mu/L)^{-1}\right)
===============================

-\log\left(1-\frac{\mu}{L}\right)
\approx
\frac{\mu}{L}.
]

所以迭代次数大约是

[
\frac{L}{\mu}
\log\left(\frac{f(x_0)-f^\ast}{\epsilon}\right).
]

这就是第 18 页中的近似式。它说明，精度 (\epsilon) 的影响是对数级别的，而条件数

[
\frac{L}{\mu}
]

是线性影响。也就是说，如果条件数变成原来的 100 倍，达到同样精度大约也会慢 100 倍。这就是为什么条件数是决定梯度下降速度的核心因素。

我们现在回到之前第 7 页的例子：

[
f(x)=\frac12\left((x^{(1)})^2+\gamma(x^{(2)})^2\right).
]

这个函数的 Hessian 是

[
\nabla^2 f(x)=
\begin{pmatrix}
1&0\
0&\gamma
\end{pmatrix}.
]

它的最小特征值是

[
\mu=1,
]

最大特征值是

[
L=\gamma.
]

所以条件数是

[
\kappa=\frac{L}{\mu}=\gamma.
]

如果 (\gamma=1)，函数等高线是圆，条件数为 1，梯度下降非常快。如果 (\gamma=10000)，条件数就是 10000，说明函数非常病态，等高线极其狭长，梯度下降会非常慢。

第 18 页还提到，在这个例子中，当

[
\gamma=10000,\qquad k=100
]

时，有

[
\left(\frac{\gamma-1}{\gamma+1}\right)^k=0.98.
]

我们来理解这个数字。此时

[
\frac{\gamma-1}{\gamma+1}
=========================

\frac{9999}{10001}
\approx 0.9998.
]

经过 100 次迭代后，

[
0.9998^{100}\approx 0.98.
]

这表示经过 100 次迭代，误差大约还剩下初始误差的 98%。也就是说，几乎没有怎么下降。这非常直观地说明了：即使函数是简单的二次函数，即使它是强凸的，梯度下降也可能因为条件数太大而非常慢。

这里要注意，第 18 页中这个 (\left(\frac{\gamma-1}{\gamma+1}\right)^k) 来自前面精确线搜索的例子；而强凸线性收敛公式里的因子是

[
1-\frac{\mu}{L}
===============

1-\frac1\gamma.
]

不同步长策略下的具体收缩因子可能不同，但它们共同表达的核心现象一样：当条件数 (\gamma) 很大时，收敛因子非常接近 1，算法就会很慢。

为什么条件数大就慢？可以从两个方向理解。一个方向是代数上的：收敛因子

[
1-\frac{\mu}{L}
]

当 (L/\mu) 很大时非常接近 1。比如 (L/\mu=10000)，那么

[
1-\frac{\mu}{L}=0.9999.
]

每一步只把误差乘以 (0.9999)，下降极慢。

另一个方向是几何上的：(L) 代表最陡方向的曲率，(\mu) 代表最平方向的曲率。如果 (L/\mu) 很大，函数就像一个狭长山谷。为了避免在陡峭方向上发散，步长必须受 (1/L) 限制；但沿着平坦方向，梯度本来就小，再乘以这么小的步长，前进就非常慢。这个几何直觉正是第 19 和第 20 页要进一步讲的内容。

所以第 18 页的结论可以概括为：对于强凸光滑函数，梯度下降虽然有线性收敛，但线性收敛的快慢取决于条件数

[
\kappa=\frac{L}{\mu}.
]

条件数越接近 1，函数越像圆碗，收敛越快；条件数越大，函数越像狭长山谷，收敛越慢。第 18 页最后一句话说 gradient method can be very slow, and very much dependent on scaling，意思就是梯度下降对变量缩放非常敏感。如果不同变量的尺度差异很大，条件数会变坏，算法就会明显变慢。

接下来第 19 页会用图像和直觉解释条件数的几何含义：为什么圆形等高线快，狭长椭圆慢。


这很好理解，请继续。


我们继续第 19 页和第 20 页。这两页其实是在把第 18 页的条件数公式变成几何直觉。第 18 页告诉我们，梯度下降的速度主要由

[
\kappa=\frac{L}{\mu}
]

决定。第 19 页进一步解释：这个 (\kappa) 不是一个抽象数字，它直接决定函数等高线的形状。第 20 页则解释为什么这种形状会拖慢梯度下降。

先看第 19 页。讲义说，条件数决定函数等高线的形状。如果

[
\kappa\approx 1,
]

也就是

[
L\approx \mu,
]

那么函数在各个方向上的曲率差不多。几何上，等高线接近圆形，函数像一个比较圆的碗。此时从任意点出发，负梯度方向大致会指向碗底，也就是最优点方向。因此梯度下降走得很直接，收敛很快。

用二次函数来理解最清楚。考虑

[
f(x)=\frac12 x^TAx.
]

如果

[
A=I,
]

那么

[
f(x)=\frac12|x|^2.
]

它的等高线满足

[
\frac12|x|^2=c,
]

也就是圆或高维球面。梯度是

[
\nabla f(x)=x.
]

负梯度方向是

[
-\nabla f(x)=-x.
]

这正好从当前点指向原点，所以梯度下降会直接朝最优点走。如果步长取 (t=1)，甚至一步就到达最优点：

[
x_{k+1}=x_k-x_k=0.
]

这就是条件数接近 1 时的理想情况。

相反，如果

[
\kappa\gg 1,
]

也就是

[
L\gg \mu,
]

函数在不同方向上的曲率差异很大。一个方向很陡，另一个方向很平。几何上，等高线会变成非常狭长的椭圆，函数像一个又窄又长的山谷。此时负梯度方向往往不是沿着谷底通向最优点，而是更偏向指向谷壁。于是梯度下降会在谷地两侧来回折返，产生我们第 8 页已经见过的之字形运动。

仍然用二次函数举例：

[
f(x,y)=100x^2+y^2.
]

等高线满足

[
100x^2+y^2=c.
]

这是一个狭长椭圆。因为 (x) 方向前面的系数是 (100)，所以 (x) 方向非常陡；而 (y) 方向相对平缓。梯度是

[
\nabla f(x,y)=
\begin{pmatrix}
200x\
2y
\end{pmatrix}.
]

即使 (x) 不大，(200x) 也可能很显著，所以梯度方向很容易被陡峭的 (x) 方向支配。结果是，负梯度方向会倾向于横向穿过山谷，而不是顺着山谷向最优点推进。

第 19 页最后问了一个问题：为什么这种几何形状会决定速度？第 20 页就是这个问题的答案。

第 20 页首先讲 curvature disparity，也就是曲率差异。在狭长山谷里，曲率非常不平衡。横穿山谷的方向非常陡，对应大曲率，也就是 (L)。沿着谷底方向非常平，对应小曲率，也就是 (\mu)。所以 (L/\mu) 很大，本质上就是“陡的方向太陡，平的方向太平”。

这个不平衡本身还不是全部问题。真正的问题在于标准梯度下降使用单一全局步长

[
t.
]

也就是说，同一步更新

[
x_{k+1}=x_k-t\nabla f(x_k)
]

中的 (t) 同时作用在所有方向上。它不能做到在陡峭方向上用小步长，在平坦方向上用大步长。可是一个理想算法恰恰应该这么做：面对陡峭谷壁时谨慎一点，面对平坦谷底时大胆一点。

标准梯度下降做不到这一点，所以就出现第 20 页说的核心冲突：稳定性与进展之间的冲突。

先看稳定性。为了避免在陡峭方向上发散，步长必须足够小。对于 (L)-smooth 函数，我们前面由充分下降引理知道，常数步长需要满足大致

[
t<\frac{2}{L}
]

才能保证下降。实际中常用更保守的

[
t=\frac1L.
]

这就是第 20 页所说的 “Wall’s Warning”：陡峭谷壁提醒你，如果步长太大，你会飞出山谷，甚至发散。因此 (L) 越大，允许的步长越小。

但是再看平坦方向。在沿着谷底通向最优点的方向上，曲率大约是 (\mu)，梯度变化很慢，梯度本身也相对弱。更新量大致是

[
t\nabla f(x).
]

如果沿谷底方向的梯度量级受 (\mu) 控制，而步长又被迫取到大约 (1/L)，那么沿谷底方向每一步的有效推进量就带有比例

[
\frac{\mu}{L}.
]

当

[
\frac{\mu}{L}
]

很小时，每一步沿真正需要前进的方向走得非常少。于是算法虽然没有发散，但进展极慢。这就是第 20 页所说的 “Floor’s Frustration”：谷底方向太平，而步长又因为谷壁太陡被限制得太小，所以向最优点推进很慢。

我们可以用一个一维分解来更精确地理解。对于二次函数

[
f(x)=\frac12 x^TAx,
]

假设 (A) 可以对角化，沿每个特征方向的曲率是 (\lambda_i)。梯度下降取步长 (t=1/L) 时，在第 (i) 个特征方向上的误差会乘以

[
1-\frac{\lambda_i}{L}.
]

如果某个方向是最陡方向，(\lambda_i=L)，那么这个方向的误差因子是

[
1-\frac{L}{L}=0,
]

也就是说这个方向消得很快。可是如果某个方向是最平方向，(\lambda_i=\mu)，那么误差因子是

[
1-\frac{\mu}{L}.
]

当 (L/\mu) 很大时，这个数非常接近 1，所以该方向的误差衰减极慢。最后整体收敛速度就被最慢的平坦方向拖住。

这也解释了为什么第 18 页的线性收敛因子是

[
1-\frac{\mu}{L}.
]

它实际上对应的是最难下降的方向，也就是曲率最小的方向。梯度下降不是每个方向都慢，而是被最慢方向支配。条件数越大，最慢方向越慢。

第 20 页最后说，结果是需要

[
O\left(\frac{L}{\mu}\right)
]

量级的迭代才能逃出山谷。更完整地说，如果要达到误差 (\epsilon)，迭代次数大约是

[
O\left(\frac{L}{\mu}\log\frac{1}{\epsilon}\right).
]

这里

[
\frac{L}{\mu}
]

来自条件数，

[
\log\frac{1}{\epsilon}
]

来自线性收敛的几何衰减。

现在我们把第 19、20 页放在一起理解。条件数小，等高线接近圆，负梯度方向比较直接地指向最优点，算法快。条件数大，等高线狭长，负梯度方向容易指向谷壁，算法之字形前进。标准梯度下降只有一个统一步长，为了不在陡方向发散，只能把步长设小；但这个小步长又导致平坦方向前进缓慢。因此条件数越大，梯度下降越慢。

这也为第 21 页的预条件梯度下降做铺垫。既然慢的根源是狭长山谷和尺度不均衡，那么自然会问：能不能把山谷“拉伸”成圆碗？能不能让不同方向使用不同尺度的步长？第 21 页的预条件矩阵 (P) 正是为了解决这个问题。

我们进入第 21 页，讲 Preconditioned Gradient Descent，也就是预条件梯度下降。它的出发点正是前面两页的问题：如果梯度下降慢，是因为函数等高线太狭长、条件数太大，那么能不能通过某种变换，把这个狭长山谷变得更圆一些？

普通梯度下降是

[
x_{k+1}=x_k-t_k\nabla f(x_k).
]

它有一个很强的限制：所有坐标方向都使用同一个步长 (t_k)。如果某些方向很陡，某些方向很平，单一步长就很尴尬。为了在陡方向上稳定，步长必须小；但这个小步长又会让平坦方向进展很慢。预条件法的思想就是：不要只用一个标量步长控制所有方向，而是先用一个矩阵 (P) 去重新缩放梯度方向。讲义给出的更新公式是

[
x_{k+1}=x_k-t_kP\nabla f(x_k),
]

其中 (P\succ0)，也就是说 (P) 是对称正定矩阵。

这里最重要的是理解 (P) 的作用。普通梯度下降方向是

[
-\nabla f(x_k).
]

预条件梯度下降方向是

[
-P\nabla f(x_k).
]

也就是说，梯度向量先经过矩阵 (P) 的变换，再作为下降方向。这个矩阵可以在不同方向上放大或缩小梯度分量。对于陡峭方向，我们希望走得小一点；对于平坦方向，我们希望走得大一点。因此 (P) 理想上应该近似“曲率的倒数”。

为什么要求 (P\succ0)？因为我们希望

[
-P\nabla f(x_k)
]

仍然是下降方向。检查一下下降方向条件：

[
p_k^T\nabla f(x_k)<0.
]

现在

[
p_k=-P\nabla f(x_k).
]

于是

[
p_k^T\nabla f(x_k)
==================

# (-P\nabla f(x_k))^T\nabla f(x_k)

-\nabla f(x_k)^TP\nabla f(x_k).
]

由于 (P\succ0)，只要

[
\nabla f(x_k)\ne0,
]

就有

[
\nabla f(x_k)^TP\nabla f(x_k)>0.
]

因此

[
-\nabla f(x_k)^TP\nabla f(x_k)<0.
]

所以 (-P\nabla f(x_k)) 确实是下降方向。这就是 (P) 要对称正定的根本原因：它既改变方向，又保证仍然下降。

我们用一个二次函数来看预条件为什么能改善条件数。考虑

[
f(x)=\frac12x^TAx,
]

其中 (A\succ0)。普通梯度是

[
\nabla f(x)=Ax.
]

普通梯度下降为

[
x_{k+1}=x_k-tAx_k=(I-tA)x_k.
]

收敛速度受 (A) 的条件数

[
\kappa(A)=\frac{\lambda_{\max}(A)}{\lambda_{\min}(A)}
]

控制。

如果使用预条件矩阵 (P)，则

[
x_{k+1}=x_k-tPAx_k=(I-tPA)x_k.
]

这时速度不再直接由 (A) 决定，而由预条件后的矩阵 (PA) 的谱性质决定。理想情况下，如果我们能取

[
P=A^{-1},
]

那么

[
PA=I.
]

于是更新变成

[
x_{k+1}=x_k-tx_k=(1-t)x_k.
]

如果取

[
t=1,
]

则

[
x_{k+1}=0,
]

一步到达最优点。这说明最理想的预条件矩阵就是 Hessian 的逆，因为它能够完全消除不同方向曲率不均衡的问题。

当然，实际中直接用 (A^{-1}) 或 Hessian 逆往往太贵，所以第 21 页给了两类常见选择。

第一类是对角矩阵，也叫 Jacobi preconditioner：

[
P=\operatorname{diag}(d_1,\ldots,d_n).
]

这种方法只对每个坐标方向分别缩放，不处理坐标之间的耦合。讲义给出的经验规则是

[
d_i=\left(\frac{\partial^2 f}{\partial x_i^2}\right)^{-1}.
]

也就是说，第 (i) 个方向的缩放因子取该方向二阶导数的倒数。二阶导数大，说明这个方向陡，(d_i) 就小；二阶导数小，说明这个方向平，(d_i) 就大。这样更新时，陡方向自动小步走，平方向自动大步走。

用前面的例子

[
f(x,y)=100x^2+y^2
]

来看。它的 Hessian 是

[
\nabla^2 f(x,y)=
\begin{pmatrix}
200&0\
0&2
\end{pmatrix}.
]

普通梯度是

[
\nabla f(x,y)=
\begin{pmatrix}
200x\
2y
\end{pmatrix}.
]

如果取 Jacobi 预条件矩阵

[
P=
\begin{pmatrix}
1/200&0\
0&1/2
\end{pmatrix},
]

那么

[
P\nabla f(x,y)
==============

\begin{pmatrix}
1/200&0\
0&1/2
\end{pmatrix}
\begin{pmatrix}
200x\
2y
\end{pmatrix}
=============

\begin{pmatrix}
x\
y
\end{pmatrix}.
]

于是预条件梯度方向变成

[
-P\nabla f(x,y)=
\begin{pmatrix}
-x\
-y
\end{pmatrix}.
]

这就直接指向原点了。也就是说，通过按曲率倒数缩放，原本狭长的椭圆问题被变成了类似圆碗的问题，之字形运动大大减少。

第二类选择是 Hessian 逆：

[
P=(\nabla^2 f(x_k))^{-1}.
]

这就是讲义说的 Newton’s Method 的预告。因为如果在每一步取

[
P=(\nabla^2 f(x_k))^{-1},
]

那么更新变成

[
x_{k+1}
=======

x_k-t_k(\nabla^2 f(x_k))^{-1}\nabla f(x_k).
]

当

[
t_k=1
]

时，就是经典牛顿法：

[
x_{k+1}
=======

x_k-(\nabla^2 f(x_k))^{-1}\nabla f(x_k).
]

牛顿法之所以通常比梯度下降快，就是因为它不仅使用梯度，还使用 Hessian 来校正不同方向的曲率。普通梯度下降只知道“往哪里下降最快”，但不知道地形弯曲程度；牛顿法通过 Hessian 逆，把陡方向压小，把平方向放大，从而更直接地朝最优点前进。

不过要注意，预条件并不是免费午餐。使用更好的 (P) 通常意味着更高的计算成本。对角预条件便宜，但只能修正坐标尺度，不能处理变量之间强耦合。Hessian 逆很强，但计算 Hessian、存储 Hessian、求逆或解线性方程都可能非常昂贵。尤其在高维问题中，完整 Hessian 是 (n\times n) 矩阵，如果 (n) 很大，直接处理几乎不可行。因此实际算法中经常会折中，例如使用近似 Hessian、拟牛顿方法、对角自适应方法等。

从几何角度看，预条件法可以理解为改变空间中的“距离度量”。普通梯度下降默认使用欧氏距离，所以“最速下降方向”是负梯度。而预条件梯度下降相当于在一个由 (P) 或 (P^{-1}) 定义的新尺度下下降。在这个新尺度里，原本狭长的等高线可能变得更圆，因此下降方向更有效。

所以第 21 页的核心思想是：梯度下降慢的根源常常是不同方向曲率差异太大；预条件法通过矩阵 (P) 重新缩放梯度，使陡方向走小步、平方向走大步，从而改善条件数。最简单的 (P) 是对角预条件，使用每个坐标方向曲率的倒数；最理想但昂贵的 (P) 是 Hessian 逆，它直接导向牛顿法。

如果你记住一句话，就是：预条件法不是改变目标函数的最优点，而是改变算法看待空间的尺度，让原本狭长的山谷变得更像圆碗，从而让梯度下降走得更直接。

好，我们继续第 22 页。前面讲的是梯度下降法本身以及它在一般光滑、凸、强凸情形下的收敛结论。第 22 页开始，讲义把视角稍微抽象出来，专门讨论“收敛速度”这个概念。也就是说，我们不只关心算法会不会收敛，还关心误差 (e_k) 是以什么速度变小的。

先看第 22 页最上面的 problem error measurements，也就是“误差如何度量”。优化算法中有好几种常见误差，不同分析会选择不同指标。第一种是点误差：

[
e_k=|x_k-x^\ast|.
]

它衡量当前迭代点 (x_k) 和真实最优解 (x^\ast) 之间的距离。这是最直观的误差，但实际中有一个问题：我们通常不知道 (x^\ast)，所以这个量更多用于理论分析，而不是实际停止准则。

第二种是函数值误差：

[
e_k=f(x_k)-f(x^\ast).
]

它衡量当前函数值距离最优函数值还有多远。这个指标在凸优化中非常常用，因为凸函数下 (f(x_k)-f(x^\ast)\ge 0)，并且它直接反映目标优化得有多好。不过实际中我们也常常不知道 (f(x^\ast))，所以仍然更多是理论量。

第三种是梯度范数：

[
e_k=|\nabla f(x_k)|.
]

它衡量当前点是否接近驻点。这个指标在实际中更容易计算，因为只要能算梯度，就能算它。对于无约束优化问题，如果 (\nabla f(x_k)) 很小，我们就认为 (x_k) 接近一个一阶稳定点。前面第 15 页的一般非凸收敛分析，用的就是梯度范数作为误差指标。

所以同一个算法，可能根据不同误差指标呈现不同的收敛表述。例如一般非凸情形下，我们常说

[
|\nabla f(x_k)|\to 0,
]

而强凸情形下，我们常说

[
f(x_k)-f^\ast\to 0
]

或者

[
|x_k-x^\ast|\to 0.
]

第 22 页接着写 convergence，也就是收敛本身。常见说法包括

[
\lim_{k\to\infty}x_k=x^\ast
]

或者

[
\lim_{k\to\infty}f(x_k)=f(x^\ast).
]

前者叫迭代点收敛到最优解，后者叫函数值收敛到最优值。点收敛通常更强，因为如果 (x_k\to x^\ast)，在 (f) 连续时自然有 (f(x_k)\to f(x^\ast))。但反过来不一定成立。函数值接近最优，并不总是意味着点本身非常接近某个特定最优解，尤其当最优解不唯一时更明显。

举个简单例子，若

[
f(x,y)=x^2,
]

那么所有满足 (x=0) 的点都是最优点，也就是整条 (y)-轴都是最优解。函数值 (f(x_k,y_k)\to0) 只说明 (x_k\to0)，但 (y_k) 可以不收敛。因此函数值收敛不一定推出点列收敛到某个唯一点。若函数强凸，最优解唯一，情况会更好。

然后第 22 页讲 iteration complexity，也就是迭代复杂度。它关心的是：为了让误差达到某个水平，需要多少次迭代。讲义列出了一些典型速度：

[
e_k\le O\left(\frac1k\right),
]

[
O\left(\frac1{k^2}\right),\qquad
O\left(\frac1{\sqrt{k}}\right),\qquad
O\left(\frac1{2^k}\right),\qquad
O\left(\frac1{2^{2^k}}\right).
]

这里的 (O(\cdot)) 是大 O 记号。比如

[
e_k\le O\left(\frac1k\right)
]

意思是存在常数 (C>0)，使得当 (k) 足够大时，

[
e_k\le \frac{C}{k}.
]

它不是说误差精确等于 (1/k)，而是说误差最多按 (1/k) 量级下降。

前面我们已经见过这些速度中的几个。一般非凸光滑梯度下降中，我们得到的是

[
\min_{i=0,\ldots,k}|\nabla f(x_i)|
\le
O\left(\frac1{\sqrt{k}}\right).
]

凸光滑梯度下降中，函数值误差是

[
f(x_k)-f^\ast\le O\left(\frac1k\right).
]

Nesterov 加速梯度法可以达到

[
O\left(\frac1{k^2}\right).
]

强凸光滑梯度下降中，函数值误差满足

[
f(x_k)-f^\ast
\le
\left(1-\frac{\mu}{L}\right)^k
(f(x_0)-f^\ast),
]

这属于线性收敛，也可以粗略理解成类似

[
O(c^k)
]

其中 (0<c<1)。讲义用

[
O\left(\frac1{2^k}\right)
]

作为线性收敛的典型代表，因为

[
\frac1{2^k}
]

每次迭代都乘以 (\frac12)，这是非常快的几何下降。

而

[
O\left(\frac1{2^{2^k}}\right)
]

对应更快的收敛，例如二次收敛的典型误差形式。直观上，如果误差每一步大致平方一次：

[
e_{k+1}\approx C e_k^2,
]

那么一旦误差小于 1，它会极快变小。比如误差从 (10^{-1}) 变成 (10^{-2})，再变成 (10^{-4})，再变成 (10^{-8})，再变成 (10^{-16})。这类速度通常出现在牛顿法在最优点附近的局部收敛分析中。

我们可以把这些速度按快慢粗略排列：

[
O\left(\frac1{\sqrt{k}}\right)
]

最慢一些，因为 (k) 增大时平方根增长慢；

[
O\left(\frac1k\right)
]

比它快；

[
O\left(\frac1{k^2}\right)
]

更快；

指数型的

[
O(c^k)
]

通常远快于任何多项式型的 (1/k^p)；

而类似

[
O\left(\frac1{2^{2^k}}\right)
]

的双指数下降则更快，常常意味着迭代次数很少就能达到极高精度。

不过这里要特别注意一个实际问题：收敛阶快，不一定总是实际运行最快。因为每次迭代的成本也很重要。比如牛顿法可能局部二次收敛，但每一步要计算 Hessian 并解线性方程；梯度下降每一步慢一些，但单步很便宜。实际算法效率取决于“迭代次数”和“每次迭代成本”的综合。

所以第 22 页的主要作用，是告诉我们以后评价算法不能只说“会收敛”，而要说明用什么误差指标、收敛到什么对象、误差随 (k) 如何变小、达到精度 (\epsilon) 需要多少次迭代。

接下来第 23 页会更精确定义几类收敛速度：Q-次线性、Q-线性、Q-超线性，以及二次收敛。这一页会从极限比值

[
\lim_{k\to\infty}\frac{e_{k+1}}{e_k}
]

出发，把“慢”“快”“非常快”用数学语言严格区分出来。

我们继续第 23 页。这一页是在第 22 页的基础上，把“收敛快慢”用更严格的数学定义表达出来。前面我们说过 (O(1/k))、(O(1/k^2))、(O(c^k)) 这些迭代复杂度形式，但第 23 页换了一个角度：假设序列 ({x_k}) 已经收敛到 (x^\ast)，然后观察相邻两次误差的比值。

设误差为

[
e_k=|x_k-x^\ast|
]

或者

[
e_k=f(x_k)-f(x^\ast).
]

第 23 页考虑极限

[
\lim_{k\to\infty}\frac{e_{k+1}}{e_k}=\mu.
]

这里讲义用了 (\mu) 表示这个极限，但要注意，它和前面强凸参数 (\mu) 只是符号相同，含义不同。这里的 (\mu) 是“相邻误差比值的极限”，不是强凸参数。为了避免混淆，你可以在心里把它叫作 (\rho)，也就是

[
\lim_{k\to\infty}\frac{e_{k+1}}{e_k}=\rho.
]

这个比值的意义很直观。它表示当 (k) 很大时，下一步误差大约是当前误差的多少倍。如果这个比例接近 1，说明每一步只减少一点点；如果这个比例是 (0.5)，说明每一步大约减半；如果这个比例趋于 0，说明误差减少得越来越猛烈。

先讲第一种情况。如果

[
\rho=1,
]

那么称为 Q-次线性收敛，也就是 Q-sublinear convergence。它的意思是误差确实趋于 0，但相邻误差之比趋近于 1。换句话说，越到后面，每一步相对于当前误差来说减少的比例越来越小，收敛比较慢。

典型例子是

[
e_k=\frac1k.
]

计算相邻比值：

[
\frac{e_{k+1}}{e_k}
===================

# \frac{1/(k+1)}{1/k}

\frac{k}{k+1}.
]

当

[
k\to\infty
]

时，

[
\frac{k}{k+1}\to 1.
]

所以 (1/k) 是次线性收敛。虽然 (1/k) 最终会趋于 0，但每一步的相对减少越来越弱。例如从 (1/100) 到 (1/101)，下降非常少。

类似地，

[
e_k=\frac1{k^2}
]

也满足

[
\frac{e_{k+1}}{e_k}
===================

# \frac{1/(k+1)^2}{1/k^2}

\left(\frac{k}{k+1}\right)^2
\to 1.
]

所以按第 23 页这个 Q-比值定义，(1/k^2) 也属于次线性收敛。它比 (1/k) 快，但仍然不是线性收敛，因为它的相邻误差比值仍然趋近于 1。这一点很容易让人误解：次线性不是说所有次线性速度都一样慢，而是说它们都没有达到几何级数那种固定比例衰减。

第二种情况是

[
0<\rho<1.
]

这叫 Q-线性收敛，也就是 Q-linear convergence。它表示当 (k) 很大时，误差每一步大约乘以一个固定的小于 1 的比例。比如

[
e_k=\left(\frac12\right)^k.
]

则

[
\frac{e_{k+1}}{e_k}
===================

# \frac{(1/2)^{k+1}}{(1/2)^k}

\frac12.
]

所以它是 Q-线性收敛，比例是 (\rho=1/2)。这类收敛通常明显快于 (1/k)、(1/k^2) 这样的多项式速度。强凸光滑函数上的梯度下降就是线性收敛，因为我们证明过

[
f(x_k)-f^\ast
\le
\left(1-\frac{\mu}{L}\right)^k(f(x_0)-f^\ast).
]

这里误差被一个几何序列控制，所以属于线性收敛类型。比例

[
1-\frac{\mu}{L}
]

越小，收敛越快；越接近 1，收敛越慢。

第三种情况是

[
\rho=0.
]

这叫 Q-超线性收敛，也就是 Q-superlinear convergence。它表示

[
\frac{e_{k+1}}{e_k}\to 0.
]

换句话说，下一步误差相对于当前误差的比例越来越小。线性收敛中，这个比例趋向某个固定数，比如 (0.5)；而超线性收敛中，这个比例本身也趋于 0，所以越到后面越快。

一个典型例子是

[
e_k=\frac{1}{2^{2^k}}.
]

它下降极快。相邻比值为

[
\frac{e_{k+1}}{e_k}
===================

# \frac{2^{-2^{k+1}}}{2^{-2^k}}

# 2^{-2^{k+1}+2^k}

2^{-2^k}
\to 0.
]

所以它是超线性收敛。牛顿法在适当条件下常常具有超线性甚至二次收敛。

第 23 页接着说，为了区分不同的超线性收敛，还要看更一般的极限：

[
\lim_{k\to\infty}\frac{e_{k+1}}{(e_k)^q}=\mu>0.
]

这里 (q>1) 时表示更高阶的收敛。如果

[
q=2,
]

那么称为二次收敛，也就是 quadratic convergence。它的意思是，当 (k) 足够大时，

[
e_{k+1}\approx \mu e_k^2.
]

这非常快。假设 (\mu) 不大，并且当前误差已经小于 1，比如

[
e_k=10^{-3},
]

那么下一步误差大约是

[
e_{k+1}\approx C\cdot 10^{-6}.
]

再下一步可能到

[
10^{-12}
]

量级。也就是说，每一步大致会把有效精度翻倍。牛顿法最著名的性质之一就是在最优点附近可能达到二次收敛。

这里我们要认真区分“线性收敛”和“二次收敛”的名字。线性收敛不是说误差图像是一条直线，而是说误差按固定比例衰减：

[
e_{k+1}\approx \rho e_k,\qquad 0<\rho<1.
]

二次收敛也不是说误差像二次函数一样下降，而是说下一步误差大约正比于当前误差的平方：

[
e_{k+1}\approx C e_k^2.
]

因为当 (e_k<1) 时，平方会变得小很多，所以二次收敛远快于线性收敛。

第 23 页脚注还提到 Q-收敛和 R-收敛。Q 的意思是 quotient，也就是通过相邻误差的商来定义，比如

[
\frac{e_{k+1}}{e_k}.
]

这是一种比较直接、严格的收敛速度定义。R 的意思是 root，它稍微宽松一些。讲义说，如果存在一个辅助序列 ({\varepsilon_k})，使得

[
|x_k-x^\ast|\le \varepsilon_k
]

对所有 (k) 成立，并且 (\varepsilon_k) Q-线性收敛到 0，那么称 ({x_k}) R-线性收敛到 (x^\ast)。也就是说，R-线性收敛允许我们不直接要求真实误差本身每一步都满足漂亮的比值关系，只要它能被一个线性收敛的上界序列控制住即可。很多优化理论中的结果其实是 R-线性的，因为我们常常证明的是误差上界按几何速度下降，而不是误差本身的相邻比值恰好有极限。

我们把第 23 页用几个典型误差序列来统一理解。如果

[
e_k=\frac1k
]

或者

[
e_k=\frac1{k^2},
]

那么

[
\frac{e_{k+1}}{e_k}\to 1,
]

所以是次线性收敛。如果

[
e_k=\left(\frac12\right)^k,
]

那么

[
\frac{e_{k+1}}{e_k}=\frac12,
]

所以是线性收敛。如果

[
e_k=\frac1{2^{2^k}},
]

那么

[
\frac{e_{k+1}}{e_k}\to0,
]

所以是超线性收敛，而且这种速度非常接近二次收敛的典型行为。

下一页第 24 页正好用图像比较这些序列：

[
a_k=\frac1{2^k},\qquad
b_k=\frac1{4^k},\qquad
c_k=\frac1{2^{2^k}},\qquad
d_k=\frac1{k+1}.

我们接下来就会用这些例子把第 23 页的定义具体化，看它们分别属于什么收敛速度，以及图中为什么有的曲线几乎瞬间贴近 0。

我们继续第 24 页。第 24 页给出四个误差序列，用来直观比较不同收敛速度：

b_k=\frac1{4^k},\qquad
c_k=\frac1{2^{2^k}},\qquad

这一页的图像就是把这些序列随着 (k) 增大而下降的速度画出来。它的目的不是引入新算法，而是帮助我们把第 23 页的 Q-线性、Q-超线性、次线性这些概念看得更具体。

先看

[
d_k=\frac1{k+1}.
]

它是典型的次线性收敛。因为

[
\frac{d_{k+1}}{d_k}
===================

# \frac{1/(k+2)}{1/(k+1)}

\frac{k+1}{k+2}
\to 1.
]

虽然 (d_k\to0)，但是相邻两项的比例越来越接近 1，说明后期每一步相对下降非常少。例如 (k) 很大时，从 (1/(k+1)) 到 (1/(k+2)) 的变化很小。因此在图中，这条曲线下降得最慢，尾部拖得最长。它代表了类似普通凸梯度下降 (O(1/k)) 的速度。

再看

[
a_k=\frac1{2^k}.
]

它是线性收敛，因为

[
\frac{a_{k+1}}{a_k}
===================

# \frac{1/2^{k+1}}{1/2^k}

\frac12.
]

这表示每一步误差都变成上一轮的一半。比如

[
1,\frac12,\frac14,\frac18,\frac1{16},\ldots

这种下降远快于 (1/(k+1))。从图像上看，(a_k) 会很快贴近 0，但不是最快的。

再看

它也是线性收敛，因为

[
\frac{b_{k+1}}{b_k}
===================

# \frac{1/4^{k+1}}{1/4^k}

\frac14.
]

所以它每一步误差变成上一轮的四分之一。它和 (a_k) 同属于 Q-线性收敛，但 (b_k) 的线性收敛因子更小，因此更快。这个例子说明，同样是线性收敛，也有快慢差别。收敛因子从 (1/2) 变成 (1/4)，速度会明显提高。

最后看

c_k=\frac1{2^{2^k}}.

它下降极快。我们列一下前几项：

[
c_0=\frac1{2^{1}}=\frac12,

[

[
c_2=\frac1{2^4}=\frac1{16},
]

[
c_3=\frac1{2^8}=\frac1{256},
]

[

[
\frac{c_{k+1}}{c_k}
===================

# \frac{2^{-2^{k+1}}}{2^{-2^k}}

# 2^{-2^{k+1}+2^k}

2^{-2^k}
\to0.
]

所以它是 Q-超线性收敛。事实上它还体现了类似二次收敛的结构。因为

[
c_{k+1}
=======

# \frac1{2^{2^{k+1}}}

# \frac1{2^{2\cdot 2^k}}

# \left(\frac1{2^{2^k}}\right)^2

c_k^2.
]

也就是说

[
c_{k+1}=c_k^2.
]

这正是二次收敛中最典型的形式

在 (C=1) 时的情况。只要误差小于 1，每次平方都会让它急剧变小。所以在第 24 页的图中，(c_k) 会非常快地贴近横轴。

这一页想表达的核心是：从“误差都会趋于 0”这个角度看，这四个序列都收敛；但从速度上看，它们差别巨大。(d_k=1/(k+1)) 是次线性，下降慢；(a_k=1/2^k) 和 (b_k=1/4^k) 是线性，但 (b_k) 更快；(c_k=1/2^{2^k}) 是超线性，而且具有二次收敛的味道，后期极快。

现在顺着讲第 25 页。第 25 页标题是 Sublinear vs Linear Convergence，也就是次线性和线性收敛的比较。它把第 24 页的图像直觉转成复杂度语言。

第 25 页说，次线性收敛，例如误差像

[
\frac{C}{j}
]

或者

[
\frac{C}{j^2}
]

那样下降，一般比较慢。这里 (j) 是迭代次数，(C) 是某个常数。如果误差满足

[
e_j\le \frac{C}{j},
]

要达到

[
e_j\le \epsilon,
]

就需要

[
\frac{C}{j}\le \epsilon.
]

解得

[
j\ge \frac{C}{\epsilon}.
]

所以迭代次数是

[
O\left(\frac{C}{\epsilon}\right).
]

如果误差满足

[
e_j\le \frac{C}{j^2},
]

要达到

[
e_j\le \epsilon,
]

需要

[
\frac{C}{j^2}\le \epsilon.
]

也就是

[
j^2\ge \frac{C}{\epsilon},
]

所以

[
j\ge \sqrt{\frac{C}{\epsilon}}.
]

因此 (1/j^2) 比 (1/j) 快很多，但它仍然属于次线性，因为相邻误差比值仍趋近于 1。

线性收敛则不同。第 25 页写，如果误差像

[
(1-\delta)^k
]

那样下降，其中

[
0<\delta<1,
]

那么要达到精度 (\epsilon)，所需迭代次数大约是

[
O\left(\frac{\log(1/\epsilon)}{\delta}\right).
]

我们推一下这个式子。假设

[
e_k\le C(1-\delta)^k.
]

要让

[
C(1-\delta)^k\le \epsilon,
]

等价于

[
(1-\delta)^k\le \frac{\epsilon}{C}.
]

取对数后得到

[
k\log(1-\delta)\le \log\left(\frac{\epsilon}{C}\right).
]

因为 (\log(1-\delta)<0)，整理后

[
k\ge
\frac{\log(C/\epsilon)}{-\log(1-\delta)}.
]

当 (\delta) 比较小时，

[
-\log(1-\delta)\approx \delta.
]

所以

[
k\approx
\frac{1}{\delta}\log\left(\frac{C}{\epsilon}\right).
]

这就是讲义中的

[
O\left(\frac{\log(1/\epsilon)}{\delta}\right).
]

这个结果说明，线性收敛对精度 (\epsilon) 的依赖是对数级别，而次线性 (1/k) 对精度的依赖是 (1/\epsilon) 级别。对小 (\epsilon) 来说，对数级别远远好于倒数级别。例如从 (10^{-3}) 提高到 (10^{-6})，次线性可能需要迭代次数增加一千倍，而线性收敛只需要增加一个常数倍数量的迭代。

不过第 25 页最后也提醒我们，不能只看大 O 形式。常数和单步成本同样重要。比如一个线性收敛算法如果每一步非常昂贵，而一个次线性算法每一步极其便宜，那么在低精度要求下，次线性算法可能更实用。现代大规模机器学习中经常出现这种情况：我们不一定需要极高精度的最优解，只需要一个足够好的低精度解；此时简单的一阶方法虽然理论速率次线性，但因为每步便宜、适合大规模数据，反而很有优势。

所以第 25 页的观点是平衡的。传统上，次线性速度常常被认为“不够有趣”，因为它比线性慢；但在现代应用中，问题规模巨大，精度需求未必很高，单步成本和可扩展性非常重要，因此次线性算法仍然非常有价值。

第 26 页主要是一张图，用来视觉比较次线性和线性收敛。虽然这一页没有额外文字，但它延续第 24、25 页的思想：线性收敛的曲线会以几何速度快速下降，而次线性曲线虽然最终也下降到 0，但尾部很长，后期改善越来越慢。你可以把第 26 页看作对第 25 页复杂度结论的图像化总结。

到这里，第 22 到第 26 页关于收敛速度分类的部分就讲完了。它和前面梯度下降分析的关系是这样的：一般非凸光滑情形下，我们得到梯度范数 (O(1/\sqrt{k}))；凸光滑情形下，函数值误差 (O(1/k))；强凸光滑情形下，函数值误差线性收敛；而未来如果学牛顿法，还会看到局部二次收敛。这些术语现在就都有了严格含义。

好，我们进入最后一部分：第 27 页和第 28 页。前面我们一直在讲“迭代次数”意义下的复杂度，比如 (O(1/k))、(O(1/\epsilon))、线性收敛等。但真正实现算法时，只知道需要多少次迭代还不够，因为每次迭代本身也有计算代价。第 27 页开始讲的就是这种“每一步要花多少计算量”的问题。

第 27 页首先定义 flop，也就是 floating-point operation，浮点运算。一个 flop 通常指两个浮点数之间的一次加法、减法、乘法或除法。比如计算

[
a+b
]

算一个 flop，计算

[
ab
]

也算一个 flop。这个计数方式当然是一种简化，因为真实计算机中加法、乘法、除法的耗时可能不同，内存访问也很重要，但在算法复杂度分析中，flop 是一个常用的基本单位。

当我们评价一个算法的计算复杂度时，通常会数总共需要多少次 flop，然后把它表示成矩阵、向量维度的函数。比如变量维度是 (n)，样本数是 (m)，矩阵大小是 (m\times n)，我们会关心计算量是 (O(mn))、(O(n^2))、(O(n^3)) 还是别的量级。

第 27 页给了一个例子：

[
m^3+3m^2n+mn+4mn^2+5m+22.
]

如果 (m,n) 都很大，复杂度分析通常只保留最高阶项，忽略低阶项和常数因子。例如这里如果不额外知道 (m) 和 (n) 谁更大，就会关注其中像

[
m^3,\quad m^2n,\quad mn^2
]

这样的高阶项，而

[
5m+22
]

通常可以忽略。若进一步知道 (m) 和 (n) 同阶，比如 (m\sim n)，那么 (m^3)、(m^2n)、(mn^2) 都是三阶量级，整体可以说是 (O(n^3))。如果知道 (m\gg n)，那么哪一项占主导可能又不同。所以复杂度不仅取决于表达式，还取决于各个维度之间的规模关系。

这和优化算法有什么关系？梯度下降每次迭代至少要计算一次梯度。理论上我们说梯度下降“每步便宜”，但便宜到什么程度，要看函数结构。比如如果目标函数涉及一个大矩阵 (A)，那么每次计算梯度可能需要矩阵向量乘法，复杂度可能是 (O(mn))。如果还要做线搜索，每尝试一个步长还可能要额外计算函数值，这些都要计入总成本。

第 27 页接着提到 line searches，也就是线搜索。线搜索中我们需要反复计算

[
f(x+td).
]

最直接做法是先形成

[
z=x+td,
]

然后计算

[
f(z).
]

形成 (z=x+td) 本身需要向量运算。如果 (x,d\in\mathbb{R}^n)，那么计算 (td) 需要大约 (n) 次乘法，再和 (x) 相加需要大约 (n) 次加法，所以是 (O(n))。然后再计算 (f(z))，具体代价取决于 (f) 的结构。

但讲义提醒我们，有时候线搜索可以利用结构来减少重复计算。比如如果目标函数是 affine composition，也就是形如

[
\phi(Ax+b),
]

那么沿方向 (d) 做线搜索时，

[
f(x+td)=\phi(A(x+td)+b).
]

展开得到

[
A(x+td)+b=Ax+b+tAd.
]

如果我们已经提前算好了

[
u=Ax+b
]

和

[
v=Ad,
]

那么每次尝试不同的 (t)，只需要算

[
u+tv
]

再代入 (\phi)，而不需要每次重新做一次矩阵向量乘法 (A(x+td))。这在回溯线搜索中尤其重要，因为回溯可能要尝试多个 (t)。如果每次都重新乘 (A)，代价会高很多；如果提前缓存 (Ax+b) 和 (Ad)，就能节省大量计算。

第 27 页还特别提到二次函数：

[
f(x)=\frac12x^TAx+b^Tx.
]

我们来看它的计算代价。若 (A\in\mathbb{R}^{n\times n}) 是稠密矩阵，那么计算

[
Ax
]

需要大约 (O(n^2)) 次运算。然后计算

[
x^TAx
]

可以先得到 (Ax)，再做内积 (x^T(Ax))，内积是 (O(n))。所以函数值计算主要由矩阵向量乘法决定，是 (O(n^2))。

它的梯度是

[
\nabla f(x)=Ax+b
]

如果 (A) 对称。计算梯度同样主要需要 (Ax)，所以也是 (O(n^2))。如果 (A) 是稀疏矩阵，只有 (N) 个非零元素，那么矩阵向量乘法可以降到 (O(N))。这就是为什么实际优化中非常重视矩阵结构：同样的数学公式，稠密和稀疏实现的成本可能差别巨大。

现在进入第 28 页的逻辑回归例子。这一页给出的是一个带 (L_2) 正则项的 logistic regression 目标函数：

[
\min_x \ell(x)
:=
\frac1m\sum_{i=1}^m
\log\left(1+\exp(-b_i a_i^Tx)\right)
+\lambda|x|^2,
]

其中标签

[
b_i\in{-1,1}.
]

这里 (a_i\in\mathbb{R}^n) 是第 (i) 个样本的特征向量，(b_i) 是它的二分类标签。变量 (x\in\mathbb{R}^n) 是模型参数。这个目标函数由两部分组成：第一部分是 logistic loss，用来衡量模型预测和标签的匹配程度；第二部分

[
\lambda|x|^2
]

是正则项，用来防止参数过大，改善泛化能力。

为了把公式写得更紧凑，讲义定义

[
p_i(x)=\frac1{1+\exp(-b_i a_i^Tx)}.
]

这个 (p_i(x)) 可以理解为模型对第 (i) 个样本“分类正确”的置信度。因为 (b_i a_i^Tx) 如果很大且为正，说明 (a_i^Tx) 的符号和标签 (b_i) 一致，分类正确且置信度高，此时

[
p_i(x)\approx 1.
]

如果 (b_i a_i^Tx) 很负，说明分类方向错了，此时

[
p_i(x)\approx 0.
]

讲义还定义

[
p(x)=(p_1(x),\ldots,p_m(x))^T.
]

并令

[
A=[a_1,\ldots,a_m]^T\in\mathbb{R}^{m\times n}.
]

注意这个写法表示 (A) 的第 (i) 行是

[
a_i^T.
]

因此

[
Ax
]

是一个 (m) 维向量，其第 (i) 个分量是

[
a_i^Tx.
]

如果令

[
b=(b_1,\ldots,b_m)^T,
]

那么

[
b\circ(Ax)
]

表示逐元素乘法，它的第 (i) 个分量是

[
[b\circ(Ax)]_i=b_i a_i^Tx.
]

所以讲义写

[
b_i a_i^Tx=[b\circ(Ax)]_i.
]

这就是把每个样本的 margin 合并成向量形式。

MATLAB 代码

[
\texttt{p=1./(1+exp(-b.*(A*x)))}
]

正是在计算整个向量 (p(x))。其中 (\texttt{A*x}) 得到所有 (a_i^Tx)，(\texttt{b.*(A*x)}) 逐元素乘上标签，(\texttt{exp}) 对每个分量取指数，最后 (\texttt{1./}) 表示逐元素取倒数。

计算 (p(x)) 的主要成本是矩阵向量乘法

[
Ax.
]

如果 (A) 是稠密的 (m\times n) 矩阵，那么这一步需要

[
O(mn)
]

次运算。后面对 (m) 个分量做逐元素乘法、指数、加法和除法，都是 (O(m))，通常低于 (O(mn))。所以整体写成 (O(mn))。如果 (A) 是稀疏矩阵，只有 (N) 个非零元素，那么计算 (Ax) 的代价是

[
O(N),
]

所以 (p(x)) 的计算也可以是 (O(N)) 量级。

接着看函数值的向量形式。原目标函数是

[
\ell(x)
=======

\frac1m\sum_{i=1}^m
\log(1+\exp(-b_i a_i^Tx))
+\lambda|x|^2.
]

因为

[
p_i(x)=\frac1{1+\exp(-b_i a_i^Tx)},
]

所以

[
\frac1{p_i(x)}
==============

1+\exp(-b_i a_i^Tx).
]

于是

[
\log(1+\exp(-b_i a_i^Tx))
=========================

# \log\left(\frac1{p_i(x)}\right)

-\log p_i(x).
]

因此损失项可以写成

[
\frac1m\sum_{i=1}^m -\log p_i(x)
================================

-\frac1m\sum_{i=1}^m \log p_i(x).
]

令

[
e=(1,1,\ldots,1)^T\in\mathbb{R}^m,
]

则

[
\sum_{i=1}^m \log p_i(x)
========================

e^T\log(p(x)),
]

其中 (\log(p(x))) 是逐元素取对数。所以函数值写成

[
\ell(x)
=======

-\frac1m e^T\log(p(x))
+\lambda|x|^2.
]

这就是讲义第 28 页给出的函数值表达式。

最后我们推导梯度公式。讲义给出的结论是

[
\nabla \ell(x)
==============

-\frac1m A^T(b\circ(e-p(x)))+2\lambda x.
]

我们逐步推导。对单个样本，定义

[
s_i=b_i a_i^Tx.
]

单个 logistic loss 是

[
\ell_i(x)=\log(1+\exp(-s_i)).
]

先对 (s_i) 求导：

[
\frac{d}{ds_i}\log(1+\exp(-s_i))
================================

\frac{1}{1+\exp(-s_i)}\cdot(-\exp(-s_i)).
]

所以

[
\frac{d\ell_i}{ds_i}
====================

-\frac{\exp(-s_i)}{1+\exp(-s_i)}.
]

注意

[
p_i(x)=\frac1{1+\exp(-s_i)}.
]

于是

[
1-p_i(x)
========

# 1-\frac1{1+\exp(-s_i)}

\frac{\exp(-s_i)}{1+\exp(-s_i)}.
]

所以

[
\frac{d\ell_i}{ds_i}=-(1-p_i(x)).
]

而

[
s_i=b_i a_i^Tx,
]

所以

[
\nabla s_i=b_i a_i.
]

由链式法则，

[
\nabla \ell_i(x)
================

-(1-p_i(x))b_i a_i.
]

对所有样本求平均：

[
\nabla\left[
\frac1m\sum_{i=1}^m \ell_i(x)
\right]
=======

-\frac1m\sum_{i=1}^m b_i(1-p_i(x))a_i.
]

现在把它写成矩阵形式。向量

[
b\circ(e-p(x))
]

的第 (i) 个分量是

[
b_i(1-p_i(x)).
]

而

[
A^T(b\circ(e-p(x)))
]

等于

[
\sum_{i=1}^m b_i(1-p_i(x))a_i.
]

因此 logistic loss 部分的梯度是

[
-\frac1m A^T(b\circ(e-p(x))).
]

正则项是

[
\lambda|x|^2=\lambda x^Tx.
]

它的梯度是

[
2\lambda x.
]

所以总梯度为

[
\nabla \ell(x)
==============

-\frac1m A^T(b\circ(e-p(x)))+2\lambda x.
]

这就得到讲义公式。

从实现复杂度看，计算这个梯度主要需要两次矩阵向量乘法。第一次是

[
Ax
]

用来计算 (p(x))，代价是 (O(mn))；第二次是

[
A^T(b\circ(e-p(x)))
]

代价也是 (O(mn))。其他逐元素操作，比如计算 sigmoid、逐元素乘法、加上 (2\lambda x)，分别是 (O(m)) 或 (O(n))，通常不是主导项。所以一次完整梯度计算的复杂度是

[
O(mn)
]

如果 (A) 稠密；如果 (A) 有 (N) 个非零元素，则是

[
O(N).
]

这就是为什么在大规模机器学习中，梯度下降或随机梯度方法很常用：它们的每步主要代价是扫描数据矩阵并做矩阵向量乘法，而不是构造和求解巨大的 Hessian 系统。

到这里，整份讲义的主体内容就讲完了。它从梯度下降的基本迭代形式开始，讲了下降方向、步长规则、充分下降引理、非凸收敛、凸收敛、强凸线性收敛、条件数、预条件、收敛速度分类，最后落到具体实现中的 flop 复杂度和逻辑回归梯度公式。第 27 到第 28 页的重点是提醒我们：理论收敛速度只是一部分，真正运行算法时还必须考虑每次迭代的计算成本。对于梯度下降，最关键的成本通常就是函数值和梯度的计算；在线搜索中，还要注意如何利用结构避免重复计算。
